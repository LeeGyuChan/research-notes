1 검토 배경 및 목적

현재 시스템은 단안 RGB 카메라 + 호모그래피(H 행렬) 방식으로 지게차와 작업자 간 실거리를 계산하고 충돌 위험을 감지한다. 테스트 영상 분석 과정에서 대각선으로 진입하는 지게차에 대해 좌·우 방향 작업자의 거리가 비대칭하게 측정되는 구조적 오차가 확인되었으며, 이 문제의 근본 원인과 업계의 실제 해결 방식을 검토하기 위해 본 문서를 작성한다.

검토 범위
분석 대상: bbox 하단 중심점 기준 거리 계산의 구조적 오차 / 카메라 위치·앙각에 따른 호모그래피 신뢰도 변화 / 업계 실제 채택 방식과 그 근거

검토 목표: 현재 시스템의 소프트웨어 개선 가능 범위 확정 + 하드웨어 구성 변경이 필요한 한계선 명시

검토 제외: 쓰러짐·시야차단 등 행동 감지 모듈 — 상위 문서(Rev.1.1)에서 별도 처리

2 핵심 문제 정의 — 대각선 배치 시 거리 비대칭 오차

테스트 영상에서 지게차가 대각선으로 진입할 때 bbox 하단 중심점이 차체의 좌측 뒷바퀴 근처에 위치하는 현상이 확인되었다. 이로 인해 동일한 실거리에 있는 좌측 작업자와 우측 작업자의 측정 거리가 다르게 계산된다.

그림 1. bbox 하단 중심점(현재) vs 하단 선분 최단거리(개선) 비교 — 대각선 지게차 시나리오

오차의 세 가지 층위

원인 A
bbox 내 빈 공간
대각선 배치 시 차체는 bbox 한쪽에 치우치고, 반대쪽은 빈 공간이 된다. 하단 중심점은 차체가 없는 영역을 가리키게 된다.
원인 B
원근 투영 왜곡
천장 카메라의 사선 앙각으로 인해 지게차 상단과 하단의 픽셀 위치가 실제 지면 위치와 다르게 투영된다. 앙각이 클수록 왜곡이 심해진다.
원인 C
호모그래피 가정 위반
H 행렬은 지면(z=0)의 점을 실좌표로 변환하는 것을 전제로 한다. 지게차 몸체의 중간·상단은 지면이 아니어서 변환 오차가 발생한다.

핵심 결론: 세 원인이 동시에 작용하기 때문에 "중심점을 여러 점으로 늘리는 멀티포인트 방식"도 근본 해결책이 되지 못한다. bbox 꼭짓점들은 여전히 지면이 아닌 공중을 가리키는 점이 포함되어 있기 때문이다. 유일하게 z=0 가정을 지키는 점은 bbox 하단 라인(y2)뿐이다.

3 방식 A — 고정 천장 카메라 + 호모그래피

현재 시스템이 채택한 방식이다. 천장 또는 고소에 카메라를 고정 설치하고 호모그래피 행렬(H)로 픽셀 좌표를 실제 지면 좌표로 변환한다.

그림 2. 카메라 앙각에 따른 bbox 왜곡 정도 — 앙각이 클수록 지게차 투영 형태 왜곡 심화
장점
인프라 변경 불필요
기존 CCTV 위치 활용 가능. 소프트웨어 중심으로 배포. 지게차 개조 없이 전체 구역 모니터링 가능. 운영 중 카메라 위치·각도 조정 용이.
단점
앙각 의존 오차 상존
카메라가 높을수록 bbox 왜곡 증가. 대각선 진입 시 거리 비대칭 오차. 지게차 높이(포크 상승 시) 변수 미반영. 캘리브레이션 주기 관리 필요.

소프트웨어 개선으로 보정 가능한 범위

# 현재 방식 fu, fv = (x1+x2)/2.0, float(y2) X, Y = pixel_to_real(fu, fv, H_mat) dist = np.hypot(wx-X, wy-Y) # 개선: 선분 최단거리 def point_to_segment_dist(p, a, b): ab = np.array(b) - np.array(a) t = np.clip(np.dot(np.array(p)-np.array(a), ab) / (np.dot(ab,ab)+1e-9), 0, 1) return float(np.linalg.norm(np.array(p) - (np.array(a)+t*ab))) left_real = pixel_to_real(x1, y2, H_mat) # bbox 좌하단 right_real = pixel_to_real(x2, y2, H_mat) # bbox 우하단 dist = point_to_segment_dist(person_real, left_real, right_real)

보정 한계: 선분 방식은 "bbox 중심점 치우침"은 해결하지만, 앙각에 의한 bbox 자체의 깊이 왜곡은 여전히 남는다. 카메라 앙각이 60° 이상인 환경에서는 소프트웨어 보정만으로 15~25% 오차가 잔류할 수 있으며, 이는 구조적 한계이다.

4 방식 B — 지게차 탑재 카메라 (업계 주류)

카메라를 지게차 본체에 직접 부착하고 온보드 프로세서에서 AI 추론을 수행한다. 카메라와 지게차의 상대 위치가 항상 고정되어 있어, 지게차 방향·각도에 무관하게 동일한 좌표계를 유지한다. 실제 상용 솔루션(OmniPro Vision AI, Proxicam, ImageVision 등)이 채택하는 방식이다.

그림 3. 지게차 탑재 4방향 카메라 구성 — 지게차 회전 방향에 무관하게 항상 동일한 시점 유지
장점
방향 독립 좌표계
카메라-지게차 상대 위치 고정. 지게차가 어느 방향을 향해도 시점 불변. 대각선 배치 오차 구조 자체가 없음. 스테레오 구성 시 깊이 직접 측정 가능.
단점
차량 개조 필요
지게차 1대당 카메라·컴퓨팅 유닛 장착. 다수 대수 현장에서 초기 비용 증가. 충격·진동에 의한 캘리브레이션 틀어짐 관리 필요. 구역 전체가 아닌 각 지게차 중심 감지.

5 방식 C — UWB · 비전 혼합

UWB(초광대역) 무선 신호로 지게차-작업자 간 거리를 직접 측정하고, 카메라 비전은 보조 수단으로 활용하는 방식이다. Litum PathAware, Pozyx 등 솔루션이 채택하며, 거리 측정을 무선 신호에 위임하므로 카메라 앙각·bbox 왜곡 문제가 원천적으로 없다.

UWB 특성
15cm 정밀도 직접 측정
벽·적재물 투과 가능. 가시선(Line-of-Sight) 불필요. 비전 카메라가 못 보는 사각지대에서도 감지. 작업자 태그 착용 또는 태그리스 선택 가능.
한계
인프라 앵커 설치 필요
현장 곳곳에 UWB 앵커 설치 필요. 전체 RTLS 구성 시 비용 증가. 작업자 태그 미착용 시 감지 불가(태그리스 모드는 정확도 낮아짐). 카메라 없이 시각 정보 부재.
혼합 전략
비전 + UWB 보완
비전: 객체 분류·행동 감지 담당. UWB: 실거리 측정 담당. 각자의 강점 분담으로 단독 운용 대비 높은 신뢰도 달성. 고위험 교차로 집중 배치 가능.

7 호모그래피 기반 거리 측정의 구조적 한계

호모그래피는 "카메라에서 바라본 지면 평면을 실제 지면 좌표로 변환"한다. 이 변환이 정확하게 성립하는 조건이 엄격하다.

그림 4. 카메라 앙각별 호모그래피 적용 가능 여부 및 권장 대응

호모그래피의 근본 가정: H 행렬은 "모든 기준점이 동일한 평면(z=0) 위에 있다"는 전제로 계산된다. 지게차처럼 높이가 있는 물체의 bbox 중간·상단 픽셀은 이 가정을 위반한다. 지면에 실제로 닿아 있는 bbox 하단 라인(y2)만이 이 가정을 만족하며, 따라서 거리 기준으로 사용할 수 있는 유일한 점/선이다.

8 bbox 기준점 오차 상세 분석

현재 방식과 개선 방식의 실제 동작 차이를 수식으로 정리한다.

현재 방식 — 단일 중심점

# get_foot() 현재 구현 def get_foot(bbox): x1, y1, x2, y2 = bbox return (x1+x2)/2.0, float(y2) # ← 항상 bbox 수평 중앙 # 문제: 지게차가 대각선이면 (cx, y2)는 차체가 없는 빈 공간을 가리킴 # → 좌측 사람: 기준점이 가까워 보임 (실제보다 짧게 측정) # → 우측 사람: 기준점이 멀어 보임 (실제보다 길게 측정)

개선 방식 — 선분 최단거리

# 지게차 하단 라인 전체를 선분으로 모델링 def calc_dist_segment(worker_real, vehicle_bbox, H_mat): x1, y1, x2, y2 = vehicle_bbox # 하단 좌·우 끝점만 변환 (y2 라인 = 지면 접지 → H 가정 성립) left_real = pixel_to_real(x1, y2, H_mat) right_real = pixel_to_real(x2, y2, H_mat) # 사람 실좌표 → 선분까지 수직 최단거리 a = np.array(left_real) b = np.array(right_real) p = np.array(worker_real) ab = b - a t = np.clip(np.dot(p - a, ab) / (np.dot(ab, ab) + 1e-9), 0.0, 1.0) closest = a + t * ab return float(np.linalg.norm(p - closest)) # 효과: 사람이 좌측이면 left_real 기준, 우측이면 right_real 기준 자동 선택 # → 대각선 배치에서 좌우 비대칭 오차 해소
YOLOv8-seg 추가 시 더 정밀한 개선 가능
bbox 대신 instance segmentation 마스크를 사용하면 차체 실루엣 하단 경계선을 접지선으로 쓸 수 있다. 대각선 배치에서 bbox보다 훨씬 정확한 차체 윤곽을 반영하며, bbox의 빈 공간 문제를 근본 해소한다. 단, YOLOv8-seg 모델로 교체 및 마스크 후처리 코드 추가가 필요하다.

9 단계별 개선 방향

즉시 적용 코드 수정 1회, 재배포
bbox 하단 선분 최단거리 방식 교체
get_foot() 함수를 calc_dist_segment()로 교체. 10줄 이내 변경으로 대각선 배치 좌우 비대칭 오차 해소. 기존 H 행렬 재사용 가능. 앙각에 의한 잔류 오차는 남으나 가장 큰 비대칭 오차는 제거된다.
단기 (1~2개월) 모델 교체 + 마스크 처리

YOLOv8n-seg 모델로 교체 후 차체 마스크 하단 경계를 접지 기준으로 사용. bbox보다 실제 차체 윤곽에 가까운 접지선 확보. 처리 속도 약 10~15% 증가 예상, GPU 가속 적용으로 보완 가능.

11 리스크 및 대응 방안

리스크내용대응 방안
앙각 잔류 오차선분 방식 적용 후에도 카메라 앙각에 의한 10~20% 오차 잔류. 실거리 2m 경보 시 실제 1.6~2.4m 편차 발생 가능.경보 임계값을 보수적으로 설정 (실측 경보거리를 요구거리보다 20~30% 크게). 현장 실측으로 앙각 추정 후 보정 계수 적용.
H 행렬 시간 경과 오차카메라 진동·이동, 건물 변형으로 H 행렬 점진적 오차 누적.분기 1회 정기 검증(기준점 실측 vs 계산값 비교). 카메라 고정 브래킷 진동 방지 처리. 오차 10% 초과 시 재캘리브레이션.
지게차 포크 상승 미반영포크가 올라간 상태에서 bbox 하단 y2는 여전히 바퀴 기준이지만 실제 위험 영역은 포크 높이까지 확장됨.현 단계에서는 바닥 충돌 위험에 집중. 포크 높이 감지는 별도 모듈(포크 keypoint 추적) 검토.
YOLO 지게차 미탐지COCO 모델에 forklift 클래스 없음. truck/car로 대체하면 오탐·미탐 혼재.지게차 전용 파인튜닝 데이터셋 구축 우선화. 단기에는 truck+car 복합 조건 + 면적 필터로 지게차 특정.
처리 지연선분 변환 추가로 프레임당 연산량 미세 증가.선분 변환은 픽셀→실좌표 변환 2회 추가이므로 실질 지연 1ms 미만. GPU 가속 환경에서 무시 가능.

상위 문서와의 연계
본 문서에서 검토한 거리 측정 개선 방안(선분 방식, YOLOv8-seg, UWB)은 상위 문서 "작업자 이상행동 감지 시스템 Rev.1.1"의 시나리오 B(지게차-작업자 직접 충돌) 모듈에 해당한다. 개선된 거리 측정값은 동일하게 TTC 계산 엔진으로 전달되며, 관심·경고·긴급 3단계 경보 체계와 연동된다. 시나리오 A(쓰러짐), C(시야차단), D(사각지대) 모듈은 본 문서 검토 범위 외이며 Rev.1.1에서 별도 처리한다.
