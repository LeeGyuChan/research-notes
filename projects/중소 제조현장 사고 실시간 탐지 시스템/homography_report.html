1 배경 및 목적

작업자 이상행동 실시간 감지 시스템의 핵심 기술 중 하나인 호모그래피 기반 실거리 변환을 실제 공장 카메라 영상으로 직접 구현하고 검증하기 위한 실습이다. 단일 카메라는 3D 공간을 2D 이미지로 투영하므로 픽셀 거리만으로는 실제 거리를 알 수 없다. 이를 해결하기 위해 캘리브레이션으로 H 행렬을 산출하고, 런타임에서 매 프레임 적용하는 파이프라인을 구축한다.

왜 픽셀 거리만으로는 안 되는가
카메라는 3D 공간을 2D로 압축하기 때문에 서로 다른 통로에 있는 두 객체가 화면에서 가까워 보일 수 있다 (오탐). 반대로 같은 바닥에 있어도 원근 효과로 멀어 보일 수 있다 (미탐). 호모그래피는 이 왜곡을 수학적으로 보정하여 픽셀 좌표를 실제 미터 단위로 변환한다.

2 핵심 개념 정리

캘리브레이션과 호모그래피의 관계

두 개념은 별개가 아니다. 캘리브레이션이 호모그래피를 만드는 과정이고, 호모그래피는 그 결과물이다.

전체 파이프라인
캘리브레이션 (1회) 기준점 픽셀 좌표 + 실측 좌표 쌍을 입력하여 H 행렬 산출. 카메라 고정 시 반영구 재사용.
→
H 행렬 (homography.json) 픽셀 (u, v) → 실제 바닥 (X, Y) 변환 공식. 3×3 행렬.
→
런타임 적용 (매 프레임) YOLO 바운딩박스 발 좌표 → H 적용 → 실거리 → TTC 판단.

발 좌표 기준 측정이 핵심인 이유

잘못된 방법
바운딩박스 중심점 사용
객체의 키나 크기가 다르면 같은 바닥 위치도 중심점 높이가 달라진다. 원근 왜곡과 결합되면 오탐이 빈번하게 발생한다.
올바른 방법
바운딩박스 하단 중심 (발/바퀴)
발과 바퀴는 항상 바닥에 닿아 있다. 키·차종과 무관하게 동일 바닥 평면 위의 한 점으로 H 행렬에 입력하면 정확한 거리가 나온다.
# 발 좌표 추출 공식 foot_u = (bbox_x1 + bbox_x2) / 2 # 수평 중심 foot_v = bbox_y2 # 하단 (발/바퀴) pt = H @ np.array([foot_u, foot_v, 1.0]) X, Y = pt[0]/pt[2], pt[1]/pt[2] # 실제 좌표 (미터)

기준점 선택 기준

우선순위기준점 종류특징
1순위기둥·벽이 바닥에 닿는 하단 모서리고정·선명·불변. 카메라에서 명확하게 식별 가능
1순위바닥 안전선(황색/백색)의 꺾임·교차점도면 있으면 줄자 없이 좌표 획득 가능
2순위바닥 타일·슬래브 줄눈 교차점규격 타일이면 한 점 실측 후 나머지 계산 가능
3순위임시 마커 (테이프·콘) 설치기존 마킹 없는 넓은 공간에 적용. 캘리브레이션 후 제거 가능
금지이동 물체 근처, 가려지는 곳측정 중 위치가 바뀌거나 픽셀 좌표가 정확하지 않음

배치 원칙: 최소 6개 이상, 화면 전체에 골고루 분산 (근거리 2점 + 중거리 2점 + 원거리 2점 이상). 한쪽에 몰리면 반대쪽 왜곡이 심해진다.

Step 2 기준점 픽셀 좌표 수집 (클릭)

Colab에서 마우스 클릭으로 픽셀 좌표를 수집하기 위해 HTML Canvas와 Colab kernel callback을 결합한 방식을 사용한다. 이미지를 base64로 인코딩하여 Canvas에 한 번만 로드하고, 클릭 시 JS가 Python 콜백을 호출하여 좌표를 수집한다.

시행착오: %matplotlib notebook / %matplotlib widget 방식과 mpl_connect 클릭 이벤트는 Colab 정적 출력에서 작동하지 않는다. Canvas 재렌더링 시 이미지가 사라지는 버그도 발생했다. 최종적으로 이미지를 JS 변수에 고정 저장하고 점만 추가 그리는 방식으로 해결했다.

# 핵심 구조: 이미지를 JS _img 변수에 한 번만 로드 _img.onload = function() { drawCanvas(); }; _img.src = 'data:image/png;base64,{IMG_B64}'; # 클릭 시 Python 콜백 호출 (이미지 재로드 없이 점만 추가) canvas.onclick = function(e) { google.colab.kernel.invokeFunction('nb.click', [cx, cy], {}); }; # Python 측: 표시 좌표 → 원본 좌표 환산 def on_canvas_click(cx, cy): rx = int(round(cx / scale)) # 축소 비율 역보정 ry = int(round(cy / scale)) clicked_pts.append([rx, ry])

클릭할 때마다 빨간 점과 번호가 찍히며 하단에 픽셀 좌표가 출력된다. 잘못 찍은 경우 [되돌리기] 버튼으로 마지막 점을 삭제할 수 있다.

Step 3 실측 좌표 입력

Step 2에서 클릭한 순서와 동일하게 줄자로 잰 실제 거리(미터)를 입력한다. 1번 기준점을 원점 (0, 0)으로 정하고, X는 좌우(가로), Y는 앞뒤(깊이) 방향으로 측정한다.

real_pts = np.array([ [ 0.0, 0.0], # 1번 원점 [ 6.0, 1.0], # 2번 — 오른쪽으로 6m, 약간 더 멀리 [-6.0, 0.5], # 3번 — 왼쪽으로 6m, 비슷한 깊이 [-3.5, 2.5], # 4번 — 왼쪽 3.5m, 앞쪽 2.5m 더 멀리 [ 5.5, -3.5], # 5번 — 오른쪽 5.5m, 카메라 방향으로 3.5m 가까움 [ 1.5, 3.5], # 6번 — 약간 오른쪽, 3.5m 더 멀리 ], dtype=np.float32)

현장 미측정 시 주의: 이번 실습에서는 현장 측정이 불가한 상황이므로 카메라 화각과 공장 규모를 고려한 어림값을 사용했다. 실제 운영 전 반드시 줄자로 재측정하고 Step 4부터 재실행해야 한다. 오차가 10% 이내가 될 때까지 반복한다.

Step 4 H 행렬 계산 (findHomography)

OpenCV findHomography()에 픽셀 좌표 배열과 실측 좌표 배열을 입력하면 3×3 변환 행렬 H가 산출된다. RANSAC 옵션으로 잘못 입력된 이상치를 자동으로 제거한다.

H_mat, mask = cv2.findHomography(pixel_pts, real_pts, cv2.RANSAC, 5.0) # H 행렬: 픽셀 (u,v) → 실제 (X,Y) 변환 공식 # mask: 각 기준점이 인라이어인지 여부 (1=정상, 0=이상치) # JSON으로 저장 — 카메라 고정 시 재사용 가능 with open('homography.json', 'w') as f: json.dump({'H': H_mat.tolist(), 'pixel_pts': pixel_pts.tolist(), 'real_pts': real_pts.tolist()}, f, indent=2)
항목설명판단 기준
Inliers 수RANSAC이 정상으로 판단한 기준점 수4개 이상이면 계산 가능. 많을수록 정확
H 행렬 값3×3 실수 행렬. 특이한 값(무한대, NaN) 없어야 함정상 범위 내 실수값
저장 파일homography.json카메라 미이동 시 재사용 가능

Step 5 발 좌표 기준 실거리 계산

바운딩박스를 직접 드래그로 그린 뒤 각 박스의 하단 중심(발/바퀴 위치)을 H 행렬에 입력하여 실거리를 계산한다. Step 2와 동일한 Canvas 방식으로 드래그 박스 그리기를 구현했다.

실거리 계산 흐름
드래그로 bbox 그리기JS mousedown/mouseup 이벤트로 좌표 수집
→
발 좌표 추출foot_u = (x1+x2)/2
foot_v = y2 (하단)
→
H 행렬 적용pt = H @ [u, v, 1]
X = pt[0]/pt[2]
→
실거리 산출√((X₁-X₂)²+(Y₁-Y₂)²)
단위: 미터
실습 결과 예시
A: (4.30, 0.27) m  |  B: (2.41, 0.47) m  |  실거리: 1.91 m
실제 시스템에서는 이 bbox를 YOLO가 자동으로 출력하며, 매 프레임 동일한 계산이 자동 실행된다.

실제 시스템과의 차이: 이번 실습에서는 바운딩박스를 수동으로 그렸다. 실제 시스템에서는 YOLO가 작업자·지게차를 자동 감지하여 bbox를 출력하고, 그 결과가 H 행렬에 자동으로 입력된다.

Step 6 Bird's-Eye View 조감도 시각화

H 행렬을 이용하여 카메라 원본 이미지를 위에서 수직으로 내려다보는 평면도(조감도)로 변환한다. 원근 왜곡이 제거되어 실제 비율의 평면도가 생성된다.

용도 1
H 행렬 검증 도구
조감도에서 기둥 간격·안전선 비율이 실제 공장 배치와 비슷하면 H 행렬이 정상. 심하게 찌그러지면 real_pts 재입력 필요.
용도 2
실시간 위치 모니터링
YOLO 연동 후 감지된 객체 위치를 조감도 위에 실시간 표시하면 관제실에서 모든 객체 위치를 한눈에 파악 가능.
PPM = 60 # 1미터 = 60픽셀로 표현 H_bev, _ = cv2.findHomography(pixel_pts, dst_pts) bev = cv2.warpPerspective(img_bgr, H_bev, (out_W, out_H)) # warpPerspective: 이미지 전체를 H 행렬로 변환하여 조감도 생성

8 다음 단계

현장 실측 및 재캘리브레이션
6개 기준점의 실제 좌표를 줄자로 측정하여 real_pts 재입력. Step 4부터 재실행하여 오차 10% 이내 달성. homography.json 최종 확정.
YOLO 모델 연동
YOLOv8/v11로 작업자·지게차를 실시간 감지하고 바운딩박스를 자동으로 H 행렬에 입력. 수동 드래그 없이 매 프레임 자동 실거리 계산.
TTC(충돌 예상 시간) 계산 연동
실거리 + 이동 벡터(MOT 추적)로 TTC 산출. TTC 3초 이하 → 경고 / 1초 이하 → 긴급 인터락 발령 로직 구현.
실시간 조감도 대시보드
YOLO 감지 결과를 Bird's-Eye View에 실시간 표시. 관제실에서 모든 객체 위치와 위험 등급을 한눈에 모니터링하는 시스템 완성.
