1 검토 배경 및 목적

본 문서는 물류·생산 혼재 사업장에서 발생하는 작업자 이상행동(쓰러짐, 충돌 위험, 시야 차단 이동, 사각지대 교차 등)을 AI 비전 기반으로 실시간 감지하기 위한 기술 구조를 검토한다. 상위 문서인 "중소 제조현장 사고 실시간 탐지 시스템 Rev.1.0"의 행동 분석 파이프라인을 상세화하는 목적으로 작성되었다.

검토 범위
감지 대상: 작업자 쓰러짐·이상 자세 / 지게차-작업자 직접 충돌 / 물체 시야 차단 후 이동 중 충돌 / 코너·교차로 사각지대 충돌

핵심 기술: YOLO 기반 객체·발 좌표 추출 / Pose Estimation 자세 분석 / 호모그래피 기반 실제 거리 변환 / TTC(충돌 예상 시간) 계산 / MOT(다중 객체 추적) 동선 예측

검토 제외: 공정 이상(U-Bolt 삽입 판정) — 별도 비전센서 파이프라인으로 처리

문서 작성 배경: 상위 문서에서 "거리·벡터 계산", "Pose Estimation"으로 기술된 행동 분석 로직이 실제 현장 환경(다양한 적재물, 지게차 차종 혼재, 넓은 공간, 카메라 높이 제약)에서 기술적으로 구현 가능한지 여부를 구체적으로 검토하고 설계 방향을 확정한다.

현장에서 발생 가능한 작업자 이상행동을 4가지 시나리오로 분류한다. 각 시나리오는 감지 메커니즘이 상이하며, 독립적으로 작동하는 서브 모듈로 구성된다.

3 시나리오 A — 쓰러짐·이상 자세 감지

쓰러짐 감지는 두 가지 방법을 병행하여 상호 보완한다. 단독 방법 사용 시 앉기·숙이기 등 정상 동작과의 오탐이 발생하며, 복합 판정으로만 신뢰도 있는 감지가 가능하다.

방법 1 — 바운딩 박스 종횡비 변화

YOLO가 출력하는 작업자 바운딩 박스의 가로(W)/세로(H) 비율을 연속 프레임에서 추적한다. 서 있을 때 비율은 0.3~0.7 수준이며, 쓰러지면 몸이 수평으로 바뀌며 2.0 이상으로 급격히 상승한다.

그림 2. 정상 자세(좌)와 쓰러진 자세(우)의 바운딩 박스 종횡비 및 keypoint 변화

단일 프레임 판정 금지: 점프·낙하 동작이나 카메라 노이즈로 인한 오탐 방지를 위해 반드시 N 프레임(권장 5~10프레임, 약 0.2~0.4초) 연속 조건을 충족해야 쓰러짐으로 확정한다.

방법 2 — Pose Estimation keypoint 분석

Pose Estimation 모델은 한 사람에서 17개 신체 keypoint(머리, 어깨, 팔꿈치, 손목, 골반, 무릎, 발목 등) 좌표를 추출한다. 쓰러짐 시 세 가지 이상 신호가 복합적으로 나타난다.

신호 1
머리-골반 Y좌표 수렴
정상 서 있음 상태에서 머리 keypoint Y좌표와 골반 keypoint Y좌표의 차이가 크다. 쓰러지면 두 좌표가 거의 같아지며 수직 신장이 소멸한다.
신호 2
하체 keypoint 소실
무릎·발목 keypoint가 카메라 시야 하단으로 사라지거나 신뢰도 점수가 임계값 이하로 하락한다. 9개 이상 keypoint 소실 시 확정 신호로 처리한다.
신호 3
어깨-골반 각도 급변
서 있을 때 어깨-골반 벡터는 수직(90°)에 가깝다. 쓰러지면 수평(0°)으로 급격히 변화한다. 각도 변화 30° 이상 + 급변 속도가 빠를 때 쓰러짐 신호로 처리한다.

4 시나리오 B — 지게차·작업자 직접 충돌 (TTC 기반)

단순 픽셀 거리가 아닌 TTC(Time to Collision, 충돌 예상 시간)를 기준으로 경보를 발령한다. TTC는 현재 속도와 방향이 유지될 경우 몇 초 후 충돌이 발생하는지를 나타내며, 픽셀 거리 기준보다 오탐률을 대폭 낮출 수 있다.

그림 3. TTC 기반 충돌 위험 감지 원리 — 상대 속도 벡터와 실거리로 충돌 예상 시간 계산

픽셀 거리만 사용할 경우의 문제: 작업자가 지게차와 같은 방향으로 이동하거나 반대 방향으로 멀어지는 경우에도 픽셀 거리가 작으면 경보가 발령된다. TTC는 상대 속도 벡터를 고려하므로 실제로 접근하는 경우만 경보를 발령한다.

TTC 계산 구현

# 발 좌표 추출 + 호모그래피 변환 → 실거리 산출
foot_u = (bbox_left + bbox_right) / 2
foot_v = bbox_bottom
real_XY = H @ [foot_u, foot_v, 1] # H: 캘리브레이션 행렬

# TTC 계산
distance = sqrt((X_fork - X_worker)² + (Y_fork - Y_worker)²)
relative_v = v_fork - v_worker # 상대 속도 벡터
closing_speed = dot(relative_v, unit_direction) # 접근 속도 (음수면 멀어지는 중)
TTC = distance / closing_speed # 충돌 예상 시간 (초), closing_speed > 0 일 때만

5 시나리오 C — 시야 차단 이동 중 충돌 감지

작업자가 박스·팔레트 등 큰 물체를 들고 이동할 때 전방 시야가 차단된 상태에서 충돌이 발생할 수 있다. 카메라에서는 작업자 상체 keypoint가 소실되고 큰 직사각형 물체가 이동하는 패턴으로 관측된다.

그림 4. 정상 작업자(좌)와 물체 운반으로 시야 차단된 작업자(우)의 keypoint 비교

감지 조건 (3가지 AND 복합)

조건 1
운반 물체 클래스 감지
YOLO가 cargo_worker 또는 object_carry 클래스를 탐지. 이 클래스는 현장 데이터로 별도 파인튜닝이 필요하며, 초기에는 수백 장 이상의 "물건을 든 작업자" 라벨 이미지로 학습한다.
조건 2
상체 keypoint 소실
Pose Estimation에서 머리·어깨·팔꿈치 keypoint(상체 기준) 중 절반 이상 소실. 단순히 몸을 돌린 경우와 물체에 가린 경우를 구별하는 핵심 조건이다.
조건 3
충돌 방향 이동 벡터
해당 작업자의 이동 벡터 방향(60° 이내)에 다른 작업자나 지게차가 존재해야 경보 발령. 단순 운반 이동과 충돌 위험 이동을 구별하는 조건이다.

초기 단계(1단계) 적용 방안: cargo_worker 라벨 데이터 수집 전까지는 "작업자 바운딩 박스 면적이 정상 대비 1.5배 이상 확장 + 상체 keypoint 소실 + 이동 중" 조건을 Rule 기반으로 적용한다. 학습 데이터 수백 장 이상 확보 후 YOLO 파인튜닝으로 전환한다.

6 시나리오 D — 코너·교차로 사각지대 충돌 예측

선반·벽 모서리에서 두 이동 객체의 동선이 교차할 위험은 카메라에 두 객체가 동시에 보이지 않을 수 있어 가장 난이도가 높은 시나리오다. MOT와 칼만 필터 기반 위치 예측으로 접근한다.

그림 5. 코너 사각지대 충돌 예측 — MOT + 칼만 필터 + 가상 위험구역 감시

1단계 대안: MOT·칼만 필터 구현 전까지는 코너·교차로에 Rule 기반 트리거 구역을 설정하여 해당 구역에 객체가 접근하면 무조건 경고를 발령하는 단순 Rule 방식을 우선 적용한다. 오탐이 많지만 안전상 허용 가능한 보수적 운영이다.

7 2D 투영의 깊이 모호성 문제

단일 카메라는 3D 공간을 2D 이미지로 투영하므로 구조적으로 깊이(depth) 정보를 잃는다. 픽셀 좌표만으로는 서로 다른 통로에 있는 객체가 가까워 보이거나(오탐), 실제로 가까운 객체가 멀어 보이는(미탐) 문제가 발생한다.

8 호모그래피 기반 실거리 변환

호모그래피(Homography)는 카메라 2D 화면의 픽셀 좌표를 실제 바닥 좌표(미터 단위)로 변환하는 3×3 행렬이다. 카메라 설치 후 1회 캘리브레이션으로 산출하며, 카메라가 움직이지 않는 한 재사용한다.

그림 7. 호모그래피 캘리브레이션 3단계 — 카메라 화면·실제 좌표·런타임 변환
발 좌표와 호모그래피는 분리 불가능한 하나의 파이프라인
발 좌표(bbox 하단 중심)는 호모그래피에 입력하는 픽셀 좌표이고, 호모그래피는 그 좌표를 실제 거리로 변환하는 도구다. "어느 픽셀을 H에 넣는가(발 좌표)"와 "H로 무엇을 하는가(거리 변환)"의 관계로, 두 가지를 분리하면 의미가 없다.

핵심: 호모그래피는 객체 크기·키·지게차 차종을 보고 거리를 추정하지 않는다. 오직 바닥 위 한 점의 픽셀 좌표를 실제 좌표로 변환할 뿐이므로, 키나 차종이 달라도 발/바퀴가 보이는 한 정확한 거리가 산출된다.

9 발 좌표 기준 거리 측정

바운딩 박스 중심점이 아닌 박스 하단 중앙(발/바퀴가 바닥에 닿는 지점)을 거리 계산의 기준점으로 사용한다. 이것이 키 차이·차종 차이와 무관하게 정확한 거리를 얻는 핵심 설계다.

그림 8. 중심점 기준(오탐, 좌)과 발 좌표 기준(정확, 우)의 거리 측정 비교
# 발 좌표 추출 (A+B 통합 파이프라인)
foot_u = (bbox_left + bbox_right) / 2
foot_v = bbox_bottom
real_XY = H @ [foot_u, foot_v, 1] # H: 캘리브레이션 행렬

# 두 객체 간 실거리 산출
dist = sqrt((X_A - X_B)**2 + (Y_A - Y_B)**2) # 단위: 미터

10 현장 캘리브레이션 절차

체커보드는 공장 현장에서 사용할 수 없다. 수십 미터 넓이의 공간에 맞는 대형 체커보드가 존재하지 않으며, 작은 체커보드는 원거리 카메라에서 식별이 불가능하다. 대신 현장에 이미 존재하는 구조물·마킹을 기준점으로 활용한다.

그림 9. 기준점 배치 비교 — 나쁜 배치(좌, 오차 30%+)와 좋은 배치(우, 오차 5~10%)

기준점으로 사용 가능한 현장 요소

1순위
바닥 안전 라인 교차점
황색·백색 안전선의 꺾임점·교차점. 직선 교차라 픽셀 클릭 정밀도가 높으며, 도면이 있을 경우 실측 없이 좌표를 획득할 수 있다.
1순위
기둥·선반 모서리 하단
구조물 모서리는 카메라에서 선명하게 구분되며 불변하는 고정점이다. 바닥 면과 만나는 하단 꼭짓점을 기준점으로 사용한다.
2순위
타일·슬래브 줄눈 교차점
규격 타일(300×300, 600×600mm)의 줄눈 교차점. 한 점만 실측하면 나머지는 타일 규격으로 계산 가능하다.
3순위
임시 마커 설치
기존 마킹이 없는 넓은 공간에 A4 용지·밝은 색 테이프·콘을 줄자로 측정하여 배치. 캘리브레이션 완료 후 제거 가능.

캘리브레이션 작업 절차

Step 1 — 현장 조사 (30분/카메라)
기준점 선택 및 실측
카메라 화각 전체에 분산된 기준점 6~10개를 선택한다. 줄자로 각 점의 실제 좌표(X, Y)를 미터 단위로 기록한다.
Step 2 — 픽셀 좌표 수집 (15분/카메라)
카메라 캡처 후 기준점 클릭
카메라 캡처 이미지를 PC에서 열고 선택한 기준점을 마우스로 클릭하여 픽셀 좌표 (u, v)를 기록한다. OpenCV 클릭 이벤트 스크립트를 활용하면 자동 좌표 출력으로 단축 가능.
Step 3 — H 행렬 계산 (5분)
OpenCV findHomography() 실행
cv2.findHomography(pixel_pts, real_pts, cv2.RANSAC) 한 줄로 3×3 변환 행렬 H 산출. RANSAC 옵션으로 오클릭 이상치 자동 제거. JSON 파일로 저장하여 재사용한다.
Step 4 — 검증 (10분)
실측값과 변환값 오차 확인
캘리브레이션에 사용하지 않은 별도 기준점 2~3개로 H 변환 결과와 실측값을 비교한다. 오차 10% 이내이면 운영 채택, 초과 시 기준점 재선택 또는 추가하여 재계산한다.

11 다중 카메라 교차 검증 (3단계 옵션)

서로 다른 각도의 카메라 2대 이상을 동기화하면 삼각측량(triangulation)으로 실제 3D 좌표를 복원할 수 있다. 단일 카메라 호모그래피보다 근본적인 해법이지만 비용과 복잡도가 높아 3단계 이후 고위험 구역 한정 적용을 권장한다.

그림 10. 다중 카메라 삼각측량으로 3D 좌표 복원 — 교차 시점이 실제 위치

12 통합 감지 파이프라인

4가지 시나리오 감지 모듈은 동일한 YOLO 추론 결과를 공유하며 병렬로 실행된다. 각 모듈의 출력은 위험 등급 판단 엔진으로 전달되어 최종 대응 등급을 결정한다.

그림 11. 통합 감지 파이프라인 — YOLO 추론 결과 공유 후 4개 모듈 병렬 실행

13 현장 조건별 구현 가능 범위 평가

100% 완벽 커버리지는 현실적으로 불가능하다. 고위험 통로와 교차로에 집중하고, 적재물 밀집 구역은 구조적 통제(바닥 마킹·펜스)로 보완하는 것이 실무에서 작동하는 전략이다.

그림 12. 현장 조건별 호모그래피 적용 가능 여부 및 대응 방안

실무 전략 요약: AI 감지는 통로·교차로 중심의 고위험 구역에 집중하고, 적재물 밀집 구역은 바닥 마킹·펜스·작업 절차로 구조적 통제를 병행한다. AI가 커버하지 못하는 영역을 구조적으로 줄이면 AI 감지 성능 요구 수준 자체가 낮아진다.

14 리스크 및 대응 방안

리스크내용대응 방안
오탐 과다앉기·숙이기·빠른 보행을 쓰러짐·충돌로 오판, 경보 남발 → 작업자 피로 및 경보 무시 습관단일 프레임 판정 금지, N 프레임 연속 조건 의무화. TTC 기반 판정으로 단순 근접 오탐 억제. 현장 시운전 후 임계값 세부 조정.
미탐적재물 가림·카메라 사각지대로 인한 실제 위험 미감지카메라 설치 전 사각지대 시뮬레이션. 고위험 구역 추가 카메라. 구조적 통제(펜스·마킹) 병행으로 AI 미감지 구역 물리 차단.
호모그래피 오차기준점 부족 또는 카메라 진동·이동으로 H 행렬 오차 누적기준점 6~10개 이상, 화각 전체 분산 배치. 분기 1회 정기 검증. 카메라 진동 방지 고정 브래킷 사용.
Pose Estimation 실패군중 밀집, 부분 가림, 낮은 해상도에서 keypoint 추출 실패Pose 단독이 아닌 종횡비와 복합 판정 설계. 신뢰도 점수 임계값 이하 시 해당 프레임 스킵 후 재판정.
시야 차단 클래스 부족cargo_worker 클래스 데이터 부족으로 초기 탐지율 낮음1단계에서 bbox 면적 팽창 + keypoint 소실 Rule 기반 대체. 데이터 누적 후 점진적 파인튜닝 전환.
Edge 연산 부하 초과다중 모듈 병렬 실행 시 20 TOPS 초과 가능성ROI 한정 추론으로 부하 완화. TensorRT FP16 변환 의무화. 부하 초과 시 Orin NX 16G(40 TOPS) 업그레이드 검토.

단계별 구현 목표

1단계 착수~개발 초기 (1~6개월)
핵심 파이프라인 구축 + Rule 기반 운영
카메라 설치 및 호모그래피 캘리브레이션 완료. YOLO 전이학습으로 작업자·지게차 탐지 구현. 쓰러짐(모듈 A) + 직접 충돌 TTC(모듈 B) 우선 적용. 코너·시야 차단은 Rule 기반 구역 감시로 대체 운영. 오탐 임계값 현장 튜닝.
2단계 개발~실증 (4~9개월)
시야 차단 감지 추가 + 데이터 누적
운반 물체 클래스(cargo_worker) 라벨 데이터 수백 장 확보 후 YOLO 파인튜닝. 시야 차단 모듈(C) 운영 전환. 위험 이벤트 데이터 누적으로 등급 분류 모델 고도화. Near-miss 데이터 기반 위험 구역 재분석.
3단계 데이터 1만 건 이상 (10~12개월)
MOT + 칼만 필터 사각지대 예측 완성
충분한 이동 궤적 데이터 기반으로 DeepSORT + 칼만 필터 구현. 사각지대 충돌 예측 모듈(D) 운영 전환. MLOps 자동 재학습 파이프라인 적용. 고위험 교차로 한정 다중 카메라 삼각측량 도입 검토. 전 모듈 통합 목표 탐지 정확도 95% 검증.
상위 문서와의 연계
본 문서에서 검토한 감지 메커니즘(시나리오 A~D)은 상위 문서 "사고 실시간 탐지 시스템 Rev.1.0"의 객체·행동 파이프라인 내 행동 분석 모듈에 해당한다. 각 모듈의 출력(이벤트 유형 + 신뢰도 점수)은 위험 등급 판단 엔진으로 전달되어 관심·경고·긴급 3단계 분류 및 경광등·인터락·관리자 알림 대응 체계와 연동된다.
