개요 논문 기본 정보 — Abstract & Introduction

Swin Transformer는 Microsoft Research Asia에서 2021년 발표한 논문으로 ICCV 2021 Best Paper Award를 수상했다. 논문 제목의 Swin은 Shifted Windows의 약자다.

논문 한 줄 요약 (Abstract)
Visual Entity의 크기 다양성과 픽셀 해상도 문제를 극복하기 위해 Shifted Windows 기반의 Hierarchical Transformer를 제안한다. Self-Attention을 비중첩 로컬 Window 내부로 제한해 이미지 크기에 선형인 계산 복잡도를 달성하면서도 Cross-Window 연결을 허용한다.
핵심 기여 ①
Hierarchical Feature Map
4개 Stage에서 H/4·H/8·H/16·H/32 해상도의 Feature Map을 생성. CNN 백본(ResNet 등)과 동일한 구조로 FPN, U-Net과 직접 연결 가능.
핵심 기여 ②
Window Self-Attention
M×M Window 내부에서만 Attention 수행. 이미지 크기에 O(N) 선형 복잡도 달성. M=7 고정 시 Window 내 연산량은 이미지 크기와 무관.
핵심 기여 ③
Shifted Window
Cyclic Shift + Masking으로 Window 경계 정보 단절 문제 해결. 계산량 증가 없이 Cross-Window 정보 교환. Sliding Window 대비 Latency 낮음.

배경 핵심 문제 정의 — ViT의 한계 (Introduction)

논문은 Transformer를 Vision에 적용할 때 두 가지 근본적인 어려움이 있다고 지적한다.

문제 1 — Visual Entity의 크기 다양성
언어 Transformer의 단어 Token과 달리 이미지의 객체 크기는 천차만별이다. Object Detection처럼 다양한 크기의 예측이 필요한 태스크에서 ViT의 단일 해상도는 치명적 약점이 된다.
문제 2 — 픽셀 해상도의 Quadratic 복잡도
ViT는 모든 Patch 쌍에 대해 Self-Attention을 수행해 계산량이 이미지 크기의 제곱에 비례한다. 픽셀 수준 Dense Prediction(Semantic Segmentation)에는 사실상 적용 불가능하다.
핵심 통찰
Window 크기 M이 고정되면 Window 내부 Attention 연산량은 이미지 크기와 무관하게 일정하다. 이미지가 커지면 Window 수만 선형으로 늘어날 뿐이라 전체 계산량이 이미지 크기에 선형 비례한다.

개념 1 Hierarchical Feature Map — 다해상도 피라미드

Hierarchical Feature Map이란 같은 이미지를 4가지 해상도로 동시에 표현한 Feature Map 피라미드다. 작은 물체는 고해상도에서, 큰 물체는 저해상도에서 잘 감지된다. 논문(Figure 1)에서는 이를 CNN의 FPN, U-Net처럼 활용할 수 있다고 설명한다.

Stage 1 — H/4×W/4
가장 세밀한 Feature Map
56×56 Token (224 입력 기준). 작은 물체(표지판, 새 등) 감지 핵심. 픽셀 디테일 보존. W-MSA + SW-MSA Block ×2 적용.
Stage 2~3
중간 크기 물체 대응
28×28, 14×14 Token. Receptive Field 확대로 더 큰 물체 감지. 채널이 깊어지며 고수준 의미(semantic) 정보를 담는다.
Stage 4 — H/32×W/32
전체 맥락 요약
7×7 Token. 이미지 전체 맥락(context) 압축. 분류 헤드(Classification Head)가 여기에 연결. ViT와 동일한 해상도.

개념 2 Window Self-Attention — 지역 Attention

Swin Transformer는 전체 이미지가 아닌 M×M 크기의 Window 내부에서만 Self-Attention을 수행한다(W-MSA). 각 Window는 독립적으로 계산되므로 복잡도가 선형이 된다.

Ω(MSA) = 4hwC² + 2(hw)²C   ← 이미지 크기 제곱에 비례
Ω(W-MSA) = 4hwC² + 2M²hwC   ← M 고정 시 이미지 크기에 선형

h, w: 패치 수 | C: 채널 수 | M: Window 크기 (기본값 7)
출처: 논문 Eq.(1), Eq.(2) — Section 3.2

CNN의 Locality Inductive Bias와 같은 원리: 이미지에서 인접한 픽셀들은 서로 높은 상관관계를 갖는다. Window 내부의 지역적 Attention으로도 충분한 시각 특징을 추출할 수 있으며, Cross-Window 정보 교환은 Shifted Window로 보완한다.

개념 3 계산 복잡도 비교 — 왜 Linear인가

이해 확인 질문
Swin은 패치를 더 잘게 쪼개서 더 많은 패치를 가지는데, 왜 계산량이 줄었다고 하는가?
답변
계산량 절감은 "작은 Patch" 때문이 아니라 "Window 내부에서만 Self-Attention"하기 때문이다. Window 크기(M=7)가 고정되므로 이미지가 커져도 각 Window 내부 연산량은 변하지 않는다. 이미지가 커지면 Window 수만 선형으로 늘어날 뿐이다.
이미지 크기ViT 계산량 O(N²)Swin 계산량 O(N)차이
64×64 (N=16 patches)16² = 2564 windows × 16 = 64유사 수준
256×256 (N=256)256² = 65,53664 windows × 49 ≈ 3,136~20× 감소
1024×1024 (N=4096)4096² ≈ 1,677만1024 windows × 49 ≈ 50,176~330× 감소
직관적 비유
ViT: 반 전체 학생이 모두 서로 대화 → 학생 수 늘면 기하급수적으로 복잡
Swin: 반을 여러 모둠으로 나누고 모둠 안에서만 대화 → 모둠 크기(M) 고정이므로 반이 커져도 선형적으로만 복잡해짐

3.1 Overall Architecture — 전체 구조

논문 Figure 3은 Swin Transformer의 전체 구조를 보여준다. Tiny 버전(Swin-T) 기준으로 4개의 Stage로 구성된다.

논문 Figure 3 — Overall Architecture 다이어그램 원본
블로그 이미지 참조 (Figure 3 전체 구조도 포함) 논문 원문 PDF Section 3.1
  1. 1
    Patch Partition 입력 RGB 이미지를 4×4 크기의 Patch로 분할한다. 하나의 Patch = 하나의 Token. 각 Patch는 4×4×3 = 48차원 벡터로 표현. Non-overlapping 분할. 224×224 이미지 → 56×56 = 3,136개 패치
  2. 2
    Stage 1 — Linear Embedding + Swin Block Linear Embedding Layer로 48 → C(=96)차원 투영. 이후 W-MSA → MLP → SW-MSA → MLP 순서의 Swin Transformer Block을 2회 통과. Activation Function으로 GELU 사용. 출력: H/4×W/4×C. Token 수 유지.
  3. 3
    Stage 2 — Patch Merging + Swin Block 인접한 2×2 패치를 Concat → 4C 차원 → Linear Layer로 2C로 축소. Token 수 ÷4, 채널 ×2. 출력: H/8×W/8×2C(=192). Block ×2.
  4. 4
    Stage 3, 4 — 반복 동일한 Patch Merging + Swin Block 반복. Stage 3: H/16×W/16×4C(=384), Block ×6. Stage 4: H/32×W/32×8C(=768), Block ×2. 4개 Stage 출력 → Hierarchical Feature Map → FPN/U-Net 연결 가능
Swin Transformer Block 내부 구조
하나의 Block은 LN → (S)W-MSA → 잔차 연결 → LN → MLP(GELU) → 잔차 연결 순서로 구성된다. 표준 Transformer Block과 동일하되 MSA를 Window 기반 MSA로 교체했다. 두 Block이 항상 쌍으로 구성되어 첫 번째는 W-MSA, 두 번째는 SW-MSA를 사용한다 (논문 Eq.3).

기법 Patch Merging — Hierarchical 구조 만들기

Patch Merging은 CNN의 Stride-2 Pooling에 대응하는 연산이다. 인접한 2×2 패치를 합쳐 해상도를 줄이고 채널을 늘린다.

3.2 Shifted Window — 경계를 넘는 Attention

W-MSA만 사용하면 Window 경계에 있는 패치들은 물리적으로 인접해도 서로 다른 Window에 속해 Attention을 수행할 수 없다. 논문은 이 문제를 Shifted Window Partitioning으로 해결한다.

W-MSA의 한계 (논문 Section 3.2)
Window 경계 패치들은 인접해도 서로 다른 Window에 속해 Attention 불가. 레이어가 쌓여도 Window 경계 너머 정보는 전파되지 않는다. "window-based self-attention module lacks connections across windows, which limits its modeling power" — 논문 원문
해결책 — Shifted Window (SW-MSA)
짝수 레이어는 W-MSA, 홀수 레이어는 SW-MSA를 번갈아 사용한다. SW-MSA에서는 Window를 (⌊M/2⌋, ⌊M/2⌋)만큼 이동(Shift)시킨 뒤 Attention을 수행한다. 두 레이어를 합치면 모든 패치가 인접 Window의 패치와 Attention할 기회를 갖는다.
l번째 Block (W-MSA):  z̃ˡ = W-MSA(LN(zˡ⁻¹)) + zˡ⁻¹  →  zˡ = MLP(LN(z̃ˡ)) + z̃ˡ
l+1번째 Block (SW-MSA): z̃ˡ⁺¹ = SW-MSA(LN(zˡ)) + zˡ  →  zˡ⁺¹ = MLP(LN(z̃ˡ⁺¹)) + z̃ˡ⁺¹

출처: 논문 Eq.(3) — Section 3.2

심화 Cyclic Shift — 구현 방법 (논문 Figure 4)

Window를 단순 이동하면 이미지 경계 밖으로 나가는 패치가 생기고 Window 수도 늘어난다. 논문은 Cyclic Shift(순환 이동) + Masking으로 이를 해결한다. 논문 Figure 4에 해당한다.

① Cyclic Shift
행·열 방향 순환 이동
전체 이미지를 M/2(=3, M=7 기준)만큼 오른쪽·아래로 순환 이동. 잘려나가는 게 아니라 반대편으로 wrap-around됨.
② Masking + Attention
유효한 쌍만 Attention
Shift로 섞인 "원래 멀리 있던 패치"끼리는 Masking(−∞ 추가)으로 차단. Window 수를 늘리지 않아 배치 연산 효율 유지.
③ Reverse Shift
원래 위치 복원
Attention 완료 후 반대 방향으로 다시 Shift해 원래 좌표계로 복원. 출력은 원본 이미지 좌표와 동일하게 정렬됨.
Cyclic Shift의 핵심 이점 (논문 Section 3.2)
Window 수 유지 + 경계 Attention 가능 = 낮은 Latency
단순 Padding 방식은 Window 수가 2×2 → 3×3으로 늘어나 2.25배 연산 증가. Cyclic Shift는 Window 수를 그대로 유지하며 Masking으로 잘못된 Attention을 차단한다. 논문 실험(Table 5, 6)에서 Sliding Window 대비 낮은 Latency와 유사한 성능을 입증.

3.2 Relative Position Bias

Swin Transformer는 절대 위치 임베딩 대신 Relative Position Bias를 Self-Attention 계산에 더해준다.

Attention(Q, K, V) = SoftMax(QKᵀ / √d + B) V

B ∈ ℝ^(M²×M²) : 상대 위치 편향 행렬
각 축 방향 상대 위치 범위 [−M+1, M−1] → 작은 편향 행렬 B̂ ∈ ℝ^(2M-1)×(2M-1)에서 참조
출처: 논문 Eq.(4) — Section 3.2
기존 절대 위치 임베딩
입력에 위치 좌표를 더함
ViT처럼 각 패치의 절대 좌표를 임베딩 벡터로 변환해 더한다. 학습 해상도와 다른 해상도에 적용 시 성능 저하.

Attention 계산 시 상대 거리를 더함

부록 GELU 활성화 함수

Swin Transformer Block의 MLP에는 ReLU 대신 GELU(Gaussian Error Linear Unit)를 사용한다. GPT, BERT, ViT 등 현대 Transformer 계열에서 표준으로 쓰이는 활성화 함수다.

GELU 근사식: f(x) = 0.5x(1 + tanh(√(2/π) · (x + 0.044715x³)))
ReLU: f(x) = max(0, x)

핵심 차이: ReLU는 x=0에서 불연속 → GELU는 x≈−0.17 부근에서 부드럽게 꺾임

활용 CNN 파이프라인 호환성 — 드롭인 교체

Swin Transformer의 실용적 핵심은 기존 CNN 백본을 그대로 교체(Drop-in replacement)할 수 있다는 점이다. 논문 Section 3.1에서 "proposed architecture can conveniently replace the backbone networks in existing methods"라고 명시하고 있다.

ResNet (CNN 백본)
특징 추출 기반 네트워크
입력 이미지에서 특징을 추출하는 기반 네트워크. Stage마다 해상도를 반씩 줄이고 채널을 2배로 늘린다. Swin도 동일한 패턴을 따르므로 교체 가능.
FPN (Object Detection용)
Feature Pyramid Network
백본의 4개 Feature Map을 위아래로 연결해 다양한 크기의 물체를 동시 탐지. H/4~H/32의 4개 출력이 필요 — Swin이 정확히 이를 충족.
U-Net (Segmentation용)
인코더-디코더 구조
인코더(백본) → Skip Connection → 디코더(업샘플링). 픽셀 단위 분류 수행. Swin의 4개 Stage 출력을 인코더로 그대로 활용 가능.
왜 ViT는 FPN에 연결하기 어려운가
FPN은 "4개의 서로 다른 해상도 Feature Map"을 받아야 작동한다. ViT는 처음부터 끝까지 H/16×W/16 단일 해상도이므로 FPN을 직접 연결할 수 없다. 억지로 연결하려면 중간 레이어를 뜯어내거나 업샘플링 모듈을 추가해야 해서 구조가 복잡해지고 성능도 떨어진다. Swin이 등장하기 전까지 Transformer가 Detection/Segmentation에서 CNN보다 실용적으로 약했던 핵심 이유다.

3.3 Architecture Variants — 모델 변형

요약 핵심 요약 — 논문의 기여와 의의

Swin Transformer 세 가지 핵심 기여
① Hierarchical Feature Map: ViT와 달리 4개의 서로 다른 해상도 Feature Map 제공. FPN, U-Net과 직접 연결 가능해 Detection/Segmentation에서 CNN 백본을 실질적으로 대체.

② 선형 계산 복잡도 O(N): Window 내부에서만 Attention해 O(N²)를 O(N)으로 감소. 고해상도 이미지에도 실용적인 계산량 유지.

③ Shifted Window로 경계 문제 해결: Cyclic Shift + Masking으로 계산량 증가 없이 Cross-Window 정보 교환. Sliding Window 대비 낮은 Latency.

이 논문에서 얻어야 할 한 가지
Swin Transformer의 진짜 기여는 "Transformer를 이미지에 쓸 수 있다"는 데서 나아가 "기존 CNN 파이프라인을 그대로 유지하면서 Transformer로 교체할 수 있다"는 실용성에 있다. Window 분할이라는 단순한 아이디어가 계산 복잡도와 Hierarchical 구조 두 문제를 동시에 해결했고, Shifted Window가 그로 인한 경계 단절을 추가 비용 없이 보완했다. 이 세 아이디어의 조합이 Vision Transformer의 실용화를 앞당겼다.
