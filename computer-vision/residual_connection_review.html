배경 소개 — 왜 깊은 네트워크가 문제인가?

신경망을 학습할 때, 우리는 보통 더 깊은 모델이 더 좋은 성능을 내리라는 기대를 갖습니다. 더 많은 layer는 더 복잡한 표현을 학습할 수 있을 것처럼 보이기 때문입니다.

하지만 현실은 다릅니다. layer를 계속 추가하여 일정 깊이에 도달한 모델은, 오히려 정확도가 감소하는 degradation 현상을 보입니다. Residual connection(잔차 연결)은 이 문제를 단순하고 효과적으로 해결하여, 현재 대부분의 CNN과 Transformer에 이르기까지 핵심 구성 요소가 되었습니다.

이 리뷰가 다루는 논문들
[1] Deep Residual Learning for Image Recognition (He et al., CVPR 2016) — ResNet 원 논문, residual connection으로 degradation 문제를 해결

[2] Residual Networks Behave Like Ensembles of Relatively Shallow Networks (Veit et al., NIPS 2016) — ResNet을 앙상블 관점으로 해석

[3] The Shattered Gradients Problem (Balduzzi et al., ICML 2017) — residual connection이 gradient 구조를 보존함을 증명

1 Residual Connection의 구조

일반적인 feed-forward 신경망에서, input은 앞에 놓인 layer들을 차례차례 통과합니다. 각 layer의 input은 이전 layer의 output이고, 다른 경로로는 layer의 건너편에 도달할 수 없습니다.

그림 1. Residual Block 구조 — F(x) + x 형태로 잔차를 학습

반면, residual connection은 "건너뛰는" 경로(skip connection)를 제공합니다. 몇 개의 layer 묶음을 하나의 block으로 볼 때, input은 일반적인 방식으로 block을 통과하는 동시에, identity mapping을 통해 단숨에 block의 끝부분으로 연결됩니다.

block의 끝부분에서 두 경로의 신호는 element-wise로 더해지고, ReLU 등의 nonlinearity를 거쳐 output으로 산출됩니다. 수식으로 표현하면 다음과 같습니다:

핵심 수식:   output = F(x) + x  →  네트워크는 full mapping H(x) 대신 잔차(residual) F(x) = H(x) − x를 학습

기존 방식 (Plain Network)
H(x) 직접 학습
input이 모든 layer를 순차적으로 통과. identity mapping을 학습하기 매우 어려움. 깊어질수록 최적화가 어려워짐.
잔차 방식 (ResNet)
F(x) = H(x) − x 학습
skip connection이 identity mapping을 직접 제공. 네트워크는 나머지 "잔차"만 학습하면 됨. 최적화가 훨씬 쉬워짐.

2 Degradation Problem

그림 2. Degradation Problem — 56-layer plain network가 20-layer보다 훈련/테스트 오차 모두 높음
핵심 질문
layer를 더 추가했는데, 왜 오히려 성능이 떨어지는가? 이것은 overfitting이나 vanishing gradient 문제인가?
답변
둘 다 아닙니다. Overfitting이라면 training error는 낮고 test error만 높아야 하는데, degradation은 training error와 test error 모두 증가합니다. Vanishing gradient는 학습이 아예 이뤄지지 않는 문제인데, degradation은 학습 자체는 이뤄지고 있습니다.

원인은 최적화의 어려움입니다. 모델이 깊어질수록 gradient descent를 이용한 최적화 자체가 지나치게 난해해지는 것입니다.
흔한 오해
더 깊은 네트워크는 항상 더 얕은 네트워크를 포함할 수 있으므로 성능이 같거나 더 좋아야 하지 않나? 이론적으로는 맞지만, 실제로는 gradient descent 기반의 최적화가 이러한 해(solution)를 찾지 못합니다.

3 Motivation — Identity Mapping 가설

논문 [1]은 하나의 직관에서 출발합니다. 더 깊은 네트워크가 그보다 얕은 네트워크 이상의 성능을 보장하는 가장 간단한 방법은?

얕은 네트워크 부분은 그대로 복사하고, 추가된 layer들을 모두 identity mapping으로 대체하면 됩니다. 어떤 이유에서인지, 우리의 깊은 네트워크는 이 단순한 구조적 해결책을 스스로 찾아내지 못합니다.

핵심 가설
Neural network는 identity mapping을 학습하는 것을 매우 어려워한다. 합성곱 연산(kernel이 전체 feature에 곱해지고 더해지는 방식)과 nonlinearity의 존재를 생각하면, input과 동일한 output을 산출하는 것은 상상 이상으로 어렵습니다.
그림 3. Residual Block — Identity mapping을 직접 제공하고 F(x)만 학습하게 함

Residual connection의 해법: identity mapping을 skip connection으로 직접 제공하고, 네트워크가 학습해야 하는 것은 그 나머지 부분인 F(x) = H(x) − x만으로 줄입니다. 만약 identity mapping이 최적에 가깝다면, F(x)를 0으로 만드는 것이 full mapping을 직접 학습하는 것보다 훨씬 쉽습니다.

핵심 포인트: 이 논문의 가설이 적중한 것인지, 이 단순한 구조적 변화는 엄청난 학습 효과의 향상을 불러왔습니다.

4 실험 결과 — ResNet의 성능

그림 4. 좌: Plain network 훈련/테스트 오차 (34-layer가 18-layer보다 높음) · 우: ResNet (34-layer가 18-layer보다 낮음)

위 그래프에서 plain network의 경우 34-layer가 18-layer보다 오차가 높은 degradation 문제를 명확히 보여줍니다. 반면 ResNet에서는 34-layer가 18-layer보다 낮은 오차를 기록하며, 깊어질수록 성능이 향상됨을 보입니다.

5 다른 해석 ① — 앙상블(Ensemble)로서의 ResNet

논문 [2]에서는 residual connection의 유효성을 앙상블(ensemble) 관점으로 설명합니다. 기존 feed-forward 신경망에서 input signal은 모든 layer를 빠짐없이 지나는 하나의 긴 경로를 따릅니다. 반면 residual connection은 signal이 각 블록을 건너뛸 수 있게 함으로써, 다양한 짧은 경로들을 제공합니다.

그림 6. Residual network를 개별 경로로 분해한 모습 — n개의 블록이 2ⁿ개의 경로를 만듦
핵심 아이디어
블록이 n개라면, skip connection이 2가지 경로를 제공하므로 전체 경로의 수는 2ⁿ개. 각 경로가 독립적으로 행동한다면?
해석
각 경로를 하나의 small network로 간주할 수 있고, 전체 ResNet은 2ⁿ개의 독립 네트워크의 ensemble처럼 기능합니다. 블록 하나를 제거해도 2ⁿ⁻¹개의 손상되지 않은 경로가 남아 있어, 기존 네트워크처럼 전체가 손상되지 않습니다.
그림 7. 좌: 경로 길이 분포 · 중: 경로 길이 당 gradient 크기 · 우: 실제 학습에 기여하는 gradient — 짧은 경로(5~17)에 집중
핵심 인사이트
학습 시 대부분의 gradient가 상대적으로 짧은 경로(path length 5~17)에 집중되어 발생합니다. 그보다 긴 경로에서는 vanishing gradient로 인해 gradient가 거의 전달되지 않습니다. 결국 ResNet은 수많은 경로 중 vanishing gradient를 피할 수 있는 짧은 경로들만 선택적으로 활용하여 학습합니다.

6 다른 해석 ② — Shattered Gradient 문제 해결

논문 [3]에서는 residual connection이 shattered gradient problem을 해결한다고 주장합니다.

Shattered Gradient란?
모델 학습 시 gradient가 white noise와 같은 형태(공간적 구조 없음)로 발생하는 문제. 뉴런의 변화가 전체 네트워크에 미치는 영향이 불안정해져 학습을 어렵게 만들며, 모델의 깊이가 깊어질수록 심각해짐.
그림 8. Gradient 패턴 비교 — (a) 1-layer: 구조적 gradient, (b) 24-layer feedforward: 점점 noise화, (c) 50-layer ResNet: 구조 유지, (d)(e) Brown/White noise 참조
기존 Deep Network
Shattered Gradient 발생
깊어질수록 gradient가 white noise에 가까워짐. Covariance matrix에서 구조적 패턴이 사라지며 학습이 불안정해짐.
50-layer ResNet
Gradient 구조 보존
Residual connection이 gradient에 공간적 구조를 추가하여 유지. Covariance matrix에서 명확한 패턴이 유지되며 학습이 안정적.

위 그림에서 1-layer feedforward는 구조적인 gradient를 가지는 반면, 24-layer feedforward는 점점 white noise에 가까워집니다. 반면 50-layer ResNet은 1-layer와 유사한 구조적 gradient를 유지하고 있음을 볼 수 있습니다.

정리 핵심 요약

결론
Residual Connection이 효과적인 3가지 이유
① Identity Mapping 제공: 학습하기 어려운 identity mapping을 skip connection으로 직접 제공하고, 네트워크는 더 학습하기 쉬운 잔차 F(x)만 집중해서 학습할 수 있게 합니다.

② Ensemble 효과: n개의 블록이 2ⁿ개의 독립 경로를 만들어 앙상블처럼 동작. 그 중 vanishing gradient를 피할 수 있는 짧은 경로들만 선택적으로 활용합니다.

③ Gradient 구조 보존: Shattered gradient 문제를 방지하여, 매우 깊은 네트워크에서도 gradient가 white noise가 되지 않고 공간적 구조를 유지합니다.

중요한 점: 우리는 여전히 residual connection에 대해 완전히 이해하고 있지는 못합니다. 그저 그것이 학습의 난이도를 크게 완화시킨다는 것을 경험적으로 알고 있을 뿐이며, 위의 해석들은 그 이유를 설명하려는 다양한 시도들입니다.
