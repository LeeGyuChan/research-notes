초록 논문 핵심 요약

논문 초록 해석
RNN에서 긴 간격의 정보를 저장하려면 역전파가 시간도 오래 걸리고, 기울기 소실 문제가 발생한다. Hochreiter(1991)의 연구를 참고하여 LSTM(Long Short-Term Memory) 방법을 고안했다. LSTM은 특별한 유닛을 통해 1,000개 이상의 타임 스텝에서도 안정적으로 오류를 제어할 수 있다. 여러 개의 게이트 유닛으로 에러 흐름을 열고 닫도록 학습하며, 복잡도는 타임 스텝과 가중치 당 O(1)이다. 기존 방법들에 비해 더 성공적이고 빠르며, 기존 RNN으로 해결할 수 없었던 시간 지연(Long-Term Dependency) 문제를 해결했다.
문제
Vanishing / Exploding Gradient
RNN의 BPTT에서 기울기가 시간 방향으로 반복 곱셈되며 소실되거나 폭발하는 문제. 장기 의존성(Long-Term Dependencies) 학습이 불가능해짐.
해결책
LSTM with Gated Units
Forget, Input, Output 게이트와 Cell State를 통해 기울기를 일정하게 유지(CEC). 1,000 타임스텝 이상의 장기 의존성도 안정적으로 학습.
성과
기존 방법 대비 우월한 성능
RTRL, BPTT 등 기존 알고리즘이 풀지 못했던 시간 지연 태스크를 성공적으로 해결. 학습 속도와 정확도 모두 향상.

배경 RNN의 한계와 LSTM의 등장

LSTM을 이해하기 위해서는 먼저 기존 RNN(Recurrent Neural Network)의 구조와 그 한계를 파악해야 한다.

RNN 기본 구조

RNN은 1980년대에 처음 제안된 구조로, 내부 메모리(hidden state)를 이용해 시퀀스 형태의 입력을 처리할 수 있다. 필기 인식, 음성 인식 등 시계열 데이터 처리에 적용된다. 각 타임 스텝에서 이전 hidden state ht-1과 현재 입력 xt를 받아 새로운 hidden state ht를 생성한다.

ht = tanh(W·xt + U·ht-1 + b)
RNN의 근본적 한계
Vanishing Gradient (기울기 소실): 역전파 과정에서 가중치 W가 계속 곱해지며, W < 1이면 기울기가 0에 수렴해 초기 타임스텝의 정보가 학습되지 않는다.

Exploding Gradient (기울기 폭발): 반대로 W > 1이면 기울기가 무한대로 발산한다. (기울기 클리핑으로 부분 해결 가능)

결과적으로 RNN은 시퀀스 간 간격(gap)이 커질수록 두 정보의 맥락 파악이 불가능해진다.
핵심 질문
RNN에서 기울기 소실이 발생하는 수학적 이유는 무엇인가?
답변
BPTT(Backpropagation Through Time)에서 ht에서 ht-k까지의 기울기는 ∂ht/∂ht-k = ∏ W 형태로 W가 k번 곱해진다. W의 최대 특이값이 1보다 작으면 지수적으로 0에 수렴(소실), 1보다 크면 지수적으로 발산(폭발)한다. LSTM은 Cell State의 덧셈(additive) 업데이트 구조를 통해 이 문제를 근본적으로 해결한다.

해결 방향: RNN에 게이트(Gate)를 추가하여 에러 흐름을 제어. 대표적 해결책이 LSTM(1997)이며, 이후 GRU(2014)가 단순화된 버전으로 등장했다.

아키텍처 LSTM 전체 구조

LSTM의 핵심 특징은 Cell State(Ct)의 존재다. 기존 RNN이 hidden state만 갖는 것과 달리, LSTM은 장기 기억(Cell State)과 단기 기억(Hidden State)을 분리하여 관리한다.

입력
xt — 현재 입력
새로 들어온 정보. 현재 타임스텝의 입력 벡터.
단기 기억
ht-1 — Hidden State
이전 타임스텝의 출력. Short-term memory 역할.
장기 기억
Ct-1 — Cell State
LSTM만의 독자적 상태. Long-term memory 역할. 게이트를 통해 선택적으로 유지·갱신.
그림 1. LSTM 상세 내부 구조 (논문 Figure 11) — Input gate, Forget gate, New memory, Output gate의 4개 모듈
그림 2. LSTM 시계열 전개 구조 — 연속된 타임스텝 t-1, t, t+1에 걸쳐 Cell State(위)와 Hidden State(아래)가 전달됨
핵심 인사이트
LSTM에는 3개의 information flow(xt, ht-1, Ct-1)와 2개의 recurrent flow(Cell State, Hidden State)가 존재한다. 4개의 게이트(Forget, Input, New Memory, Output)는 모두 뉴럴넷으로 구현되어 있으며, 어떤 정보를 기억하고 잊을지를 학습을 통해 자동으로 결정한다.

게이트 LSTM 4단계 동작 순서

LSTM은 각 타임스텝에서 다음 4단계를 순서대로 수행한다.

① Forget Gate
→
② Input Gate + New Memory
→
③ Cell State 업데이트
→
④ Output Gate

① Forget Gate — 무엇을 잊을 것인가?

ht-1과 xt를 입력으로 받아 sigmoid를 통과시킨다. 출력값 ft는 0~1 사이로, Cell State의 각 값을 얼마나 기억할지(1=완전 기억, 0=완전 삭제)를 결정한다.

ft = σ(Wf·[ht-1, xt] + bf)
그림 3. Forget Gate — sigmoid를 통해 이전 Cell State를 얼마나 보존할지 결정

② Input Gate + New Memory — 무엇을 새로 저장할 것인가?

두 부분으로 구성된다.

  • Input Gate (it): sigmoid를 통해 새 정보를 Cell State에 얼마나 반영할지 결정 (0~1)
  • New Memory (C̃t): tanh를 통해 새로운 후보 기억 생성 (-1~1)
it = σ(Wi·[ht-1, xt] + bi)     C̃t = tanh(WC·[ht-1, xt] + bC)
그림 4. Input Gate & New Memory — sigmoid(it)와 tanh(C̃t)의 원소곱으로 새 기억을 생성

③ Cell State 업데이트 — 장기 기억 갱신

Forget Gate와 Input Gate의 결과를 합산하여 새로운 Cell State Ct를 계산한다. 덧셈 연산이 핵심으로, 이것이 기울기 소실을 방지하는 핵심 메커니즘이다.

그림 5. Cell State 업데이트 수식 — ft ∘ ct-1(잊기) + it ∘ C̃t(새 기억)
그림 6. Cell State 업데이트 — 수평으로 흐르는 Cell State에 forget(×)과 input(+)이 작용함
왜 덧셈인가
곱셈만 사용하는 RNN과 달리 LSTM의 Cell State는 additive update(덧셈 갱신)를 사용한다. 이 덧셈 구조가 역전파 시 기울기를 일정하게 유지시켜(CEC: Constant Error Carousel) vanishing gradient를 방지한다.

④ Output Gate — 무엇을 출력할 것인가?

Cell State의 어떤 부분을 다음 hidden state로 노출할지 결정한다. final memory Ct와 hidden state를 분리하는 역할로, 출력에 필요 없는 장기 기억 정보를 숨긴다.

ot = σ(Wo·[ht-1, xt] + bo)     ht = ot ∘ tanh(Ct)
그림 7. Output Gate — sigmoid(ot)와 tanh(Ct)의 원소곱으로 최종 hidden state ht 생성

수식 LSTM 핵심 수식 정리

게이트 / 상태 수식 역할 활성화 함수
Forget Gate ft = σ(Wf·[ht-1, xt] + bf) 이전 기억 유지 비율 sigmoid (0~1)
Input Gate it = σ(Wi·[ht-1, xt] + bi) 새 정보 반영 비율 sigmoid (0~1)
New Memory C̃t = tanh(WC·[ht-1, xt] + bC) 새 후보 기억 생성 tanh (-1~1)
Cell State Ct = ft ∘ Ct-1 + it ∘ C̃t 장기 기억 업데이트 덧셈 (핵심!)
Output Gate ot = σ(Wo·[ht-1, xt] + bo) 출력 노출 비율 sigmoid (0~1)
Hidden State ht = ot ∘ tanh(Ct) 최종 출력 (단기 기억) tanh (-1~1)

∘ 기호는 원소곱(Hadamard product): 행렬 전체 곱이 아니라 같은 위치의 원소끼리 곱하는 연산. 각 메모리 차원을 독립적으로 조절할 수 있게 한다.

분석 Vanishing Gradient 해결 메커니즘

핵심 질문
LSTM이 RNN과 달리 vanishing gradient를 방지할 수 있는 이유는?
답변 — CEC (Constant Error Carousel)
RNN: ∂ht/∂ht-k = ∏k W — 가중치 W가 k번 곱해지므로 깊이에 따라 기울기 소실/폭발 발생.

LSTM Cell State: ∂Ct/∂Ct-1 = ft — forget gate 값(0~1)이 곱해지지만, Cell State 업데이트가 덧셈이므로 기울기가 직접 흘러갈 수 있는 경로가 확보됨. W가 반복 곱해지는 구조가 사라진다.

이를 CEC(Constant Error Carousel)라 부르며, LSTM의 핵심 아이디어다. 게이트가 4개의 컴포넌트에 대한 미분값을 각각 제공해 기울기 소실을 막는다.

비교 LSTM vs GRU

GRU(Gated Recurrent Unit, 2014)는 LSTM을 단순화한 후속 모델이다. 두 모델의 차이를 이해하면 실전 적용 시 선택 기준을 세울 수 있다.

비교 항목LSTM (1997)GRU (2014)
게이트 수 3개 (Forget, Input, Output) 2개 (Reset, Update)
State 수 2개 (Cell State + Hidden State) 1개 (Hidden State만)
Cell State 분리 있음 (장기/단기 명확히 분리) 없음 (Hidden State 하나로 통합)
파라미터 수 많음 → 학습 데이터 많이 필요 적음 → 적은 데이터로도 학습 가능
학습 속도 상대적으로 느림 빠름
성능 (충분한 데이터) 더 좋은 결과 가능 LSTM과 비슷하거나 약간 낮음
Vanishing Gradient CEC로 해결 Update Gate로 해결

실전 선택 기준: 데이터가 충분하면 LSTM, 학습 리소스가 제한적이거나 데이터가 적으면 GRU. 최근에는 두 모델 모두 Transformer에 대부분의 자리를 내주었지만, 경량 시계열 모델에서는 여전히 활발히 사용된다.

총평 핵심 요약 및 의의

결론
LSTM은 RNN의 근본적 한계를 해결한 이정표적 모델
LSTM이 제안한 Cell State + 게이트 메커니즘은 단순히 기울기 소실을 해결하는 것을 넘어, 신경망이 무엇을 기억하고 무엇을 잊을지를 스스로 학습할 수 있게 만들었다. 이 아이디어는 이후 GRU, Attention Mechanism, Transformer 등 현대 딥러닝 아키텍처의 직접적 토대가 되었다. 특히 Additive update(덧셈 갱신)를 통한 기울기 안정화 개념은 ResNet의 Skip Connection과도 맥락이 닿아 있다.
강점
장기 의존성 학습 가능
CEC 구조로 1,000 타임스텝 이상의 장기 패턴도 안정적으로 학습. 기존 RNN이 해결하지 못했던 시간 지연 문제 해결.
강점
해석 가능한 게이트
각 게이트가 "무엇을 기억/잊을/출력할지"라는 명확한 역할을 가짐. 모델 동작의 직관적 이해 가능.
한계
높은 파라미터 수
게이트 4개 구조로 GRU 대비 파라미터가 많아 학습 시간과 데이터 요구량이 크다. Transformer 등장 이후 자연어 처리에서 사용 감소.
역사적 의의
1997년 제안 당시에는 크게 주목받지 못했으나, 2010년대 딥러닝 부흥기에 음성 인식(Google, Baidu)과 자연어 처리에서 폭발적으로 사용되며 재조명받았다. Attention is All You Need(2017)의 Transformer가 등장하기 전까지 NLP의 사실상 표준 구조였으며, 현재도 경량 시계열 모델과 임베디드 환경에서 활발히 쓰인다.
