개요 논문 기본 정보

Grounding DINO는 텍스트로 지정된 임의의 물체를 이미지에서 검출하는 open-set object detection 모델이다. 핵심 아이디어는 SOTA Transformer 기반 검출기인 DINO와 언어 정보를 결합하는 grounded pre-training을 통해, 학습 시 보지 못한 새로운 카테고리의 물체도 텍스트 프롬프트만으로 검출할 수 있게 한다는 것이다.

논문의 한 줄 요약
"closed-set detector를 language-aware로 확장하되, 세 단계(neck · 쿼리 초기화 · head) 모두에서 이미지-언어 feature를 융합하면 기존 open-set 방법보다 훨씬 강력한 검출기를 만들 수 있다." COCO zero-shot에서 52.5 AP, COCO fine-tune에서 63.0 AP라는 당시 최고 성능을 달성했다.
기반 검출기
DINO (Transformer 기반)
DETR 계열의 SOTA closed-set detector. NMS 없이 end-to-end 최적화 가능. 언어 모듈과 구조적으로 친화적인 Transformer 아키텍처를 갖는다.
핵심 기여
3단계 Cross-Modality Fusion
neck(Feature Enhancer), 쿼리 초기화(Language-Guided Query Selection), head(Cross-Modality Decoder) 세 단계 모두에서 이미지-언어 feature를 융합한다.
성능 (COCO)
52.5 AP (zero-shot) · 63.0 AP (fine-tune)
COCO zero-shot 52.5 AP로 당시 1위. fine-tune 시 63.0 AP. ODinW, LVIS 등 다양한 벤치마크에서 GLIP 등 기존 방법을 상회.

배경 문제 정의 — Open-Set Detection이란 무엇인가

핵심 질문
기존 object detector는 왜 학습 시 보지 못한 카테고리의 물체를 검출하지 못하는가?

기존 Closed-Set detector(Faster-RCNN, YOLO 등)는 학습 데이터에 포함된 카테고리만 검출할 수 있다. 80개 COCO 카테고리로 학습하면 81번째 카테고리의 물체는 전혀 검출하지 못한다. 이는 classifier head가 고정된 카테고리 수를 출력하도록 설계되어 있기 때문이다.

Closed-Set Detection의 한계
새로운 카테고리를 검출하려면 해당 카테고리의 labeled 데이터를 수집하고 재학습해야 한다. 산업 현장·의료·자율주행 등 새로운 객체가 지속적으로 등장하는 환경에서는 이 비용이 감당하기 어려운 수준이다.
Open-Set Detection의 핵심 아이디어
카테고리 이름 대신 자연어(텍스트)를 검출 기준으로 사용하면, 학습 시 보지 못한 물체도 텍스트 설명만으로 검출할 수 있다. "a yellow dog wearing a red collar"처럼 속성까지 지정하는 Reference Expression Comprehension(REC)도 동일한 프레임워크로 처리 가능하다.

기존 open-set 방법인 GLIP은 1단계 검출기 Dynamic Head 위에 언어 정보를 추가했다. 하지만 closed-set detector의 성능이 곧 open-set 성능의 상한이므로, SOTA Transformer 검출기 DINO를 베이스로 삼으면 더 강력한 open-set 검출기를 만들 수 있다는 것이 본 논문의 출발점이다.

배경 개념 1 DINO — Grounding DINO의 기반 검출기

DINO(DETR with Improved deNoising anchor boxes)는 DETR 계열 Transformer 검출기 중 당시 SOTA 성능을 보인 모델이다. Grounding DINO는 DINO의 구조를 직접 확장하므로, DINO의 핵심 개념을 이해하는 것이 중요하다.

DETR 계열의 특징
NMS 없는 End-to-End 검출
기존 anchor-based 검출기는 수천 개의 proposal을 생성 후 NMS로 중복 제거한다. DINO는 이분 매칭(Hungarian algorithm)으로 NMS 없이 end-to-end 최적화. 언어 모듈과 결합하기 훨씬 단순하다.
Encoder-Decoder 구조
이미지 Feature → 쿼리 → 박스
Swin Transformer로 멀티스케일 feature 추출 → Encoder에서 feature 강화 → Decoder 쿼리가 feature를 조사해 bbox 예측. 이 layer-by-layer 구조가 언어 정보와 자연스럽게 통합된다.
Mixed Query Selection
콘텐츠 + 위치 쿼리 분리
각 디코더 쿼리는 "어디를 볼 것인가(위치)"와 "무엇을 볼 것인가(콘텐츠)" 두 파트로 구성. 위치는 encoder 출력으로 동적 초기화. Grounding DINO는 여기에 언어 정보를 추가 주입한다.
왜 DINO를 선택했는가
Transformer 기반 검출기는 언어 인코더(BERT)와 동일한 self-attention 구조를 공유한다. 따라서 이미지 feature와 텍스트 feature를 같은 메커니즘(cross-attention)으로 융합하기 자연스럽다. 반면 CNN 기반 검출기는 구조적으로 이질적이어서 언어와의 결합이 어렵다.

배경 개념 2 GLIP과 비교 — 무엇이 달라졌는가

GLIP은 Grounding DINO의 직접적인 선행 연구다. object detection을 구문 grounding 문제로 재정의하고 contrastive training을 도입했으나, 1단계 CNN 검출기(Dynamic Head) 위에 언어 정보를 부분적으로만 융합했다는 한계가 있다.

비교 항목 GLIP Grounding DINO
기반 검출기 Dynamic Head (CNN 1-stage) DINO (Transformer SOTA)
Fusion 위치 neck 일부만 neck + 쿼리 초기화 + head (3단계)
NMS 필요 여부 필요 불필요 (end-to-end)
COCO zero-shot AP 49.8 52.5 (+2.7)
REC 지원 제한적 명시적 지원
대규모 데이터 활용 보통 우수 (Transformer 특성)

핵심 차이: 저자들은 이미지-언어 feature 융합이 많을수록 성능이 높아진다고 주장한다. GLIP이 neck 단계에서만 융합했다면, Grounding DINO는 neck(Feature Enhancer), 쿼리 초기화(Language-Guided Query Selection), head(Cross-Modality Decoder) 세 단계 모두에서 융합을 수행한다. 이 tight fusion이 성능 차이의 핵심 원인이다.

아키텍처 전체 구조 — 이중 인코더 · 단일 디코더

Grounding DINO는 이중 인코더(Dual-Encoder) · 단일 디코더(Single-Decoder) 구조를 갖는다. 이미지 인코더와 텍스트 인코더가 각각 feature를 추출하고, 이를 세 단계에 걸쳐 교차 융합한 뒤 단일 디코더가 최종 bbox와 구문을 예측한다.

모듈 ① Feature Enhancer — Neck의 Cross-Modality Fusion

Feature Enhancer는 이미지 feature와 텍스트 feature를 동시에 강화하는 neck 모듈이다. 단순히 한쪽이 다른 쪽을 참조하는 것이 아니라, 이미지→텍스트, 텍스트→이미지 양방향 cross-attention이 모두 포함되어 상호 강화가 이루어진다.

이미지 Self-Attention
Deformable Self-Attention
멀티스케일 이미지 feature를 처리하는 Deformable Attention 사용. 전체 픽셀이 아닌 특정 포인트만 참조하여 계산 효율을 높인다.
Text→Image Cross-Attention
텍스트가 이미지를 강화
텍스트 feature가 query, 이미지 feature가 key/value. 이미지의 각 위치가 텍스트 토큰 중 어느 것과 관련 있는지 학습한다.
Image→Text Cross-Attention
이미지가 텍스트를 강화
이미지 feature가 query, 텍스트 feature가 key/value. 각 텍스트 토큰이 이미지의 어느 영역과 연결되는지 학습한다. GLIP에 없던 방향이다.

모듈 ② Language-Guided Query Selection — 텍스트 기반 쿼리 초기화

기존 DETR 계열 모델은 인코더 출력에서 상위 N개(900개)의 feature를 디코더 쿼리로 선택한다. Grounding DINO는 여기에 텍스트 정보를 결합하여, 입력 텍스트와 관련성이 높은 이미지 영역의 feature를 쿼리로 우선 선택한다.

Mixed Query Selection 상세
각 선택된 쿼리는 두 파트로 구성된다. 위치(Position) 파트는 인코더 출력의 해당 위치를 동적 앵커 박스로 초기화한다. 콘텐츠(Content) 파트는 학습 가능한 임베딩으로 설정된다. 이 구조는 DINO의 Mixed Query Selection을 그대로 따르되, 선택 기준에 텍스트 유사도를 추가한 것이다.
선택된 900개 쿼리 = Decoder의 Q
이 단계에서 선택된 900개의 이미지 feature가 곧 Cross-Modality Decoder의 Query(Q)가 된다. Attention 메커니즘에서 Q는 "내가 무엇을 알고 싶은가"를 나타내는 질문자다. 즉 각 쿼리는 "나는 이 이미지 영역이 텍스트의 어느 단어와 대응하고, bbox는 어디인가?"를 묻는 존재가 된다. 이후 디코더에서 이미지(K/V)와 텍스트(K/V)를 참조하며 그 질문에 스스로 답하는 과정이 반복된다.

모듈 ③ Cross-Modality Decoder — Head에서의 Fusion

Cross-Modality Decoder는 Grounding DINO에서 가장 핵심적인 구조적 변경점이다. 기존 DINO의 디코더 레이어는 Self-Attention → Image Cross-Attention → FFN 순서였다. Grounding DINO는 여기에 Text Cross-Attention을 추가하여 각 디코더 레이어가 이미지와 텍스트 양쪽을 모두 참조하며 쿼리를 정제한다.

Text Cross-Attention이 중요한 이유: 디코더 쿼리가 이미지를 참조할 때(Image Cross-Attention)는 "어디에 물체가 있는가"를 학습한다. 텍스트를 참조할 때(Text Cross-Attention)는 "이 물체가 입력 텍스트 중 어느 구문과 대응하는가"를 학습한다. 두 정보를 레이어마다 통합하므로 더 정밀한 language-aware localization이 가능하다.

심화 Q · K · V 심화 분석 — 단계별 역할 완전 정리

Attention 메커니즘의 세 요소 Q(Query), K(Key), V(Value)가 Grounding DINO의 각 단계에서 구체적으로 무엇을 가리키는지 정리한다. 같은 Cross-Attention이라는 이름이 Feature Enhancer와 Decoder 양쪽에 등장해 혼란스러울 수 있으므로, 핵심 차이인 "누가 Q인가"에 집중한다.

Cross-Modality Decoder의 각 Attention에서 Q · K · V

Attention 종류 Q (질문자) K / V (참조 대상) 이 단계에서 학습하는 것
Self-Attention 쿼리 900개 쿼리 900개 (자기 자신) 쿼리끼리 서로 참조 — "내가 보는 영역이 다른 쿼리와 겹치는가"
Image Cross-Attention 쿼리 900개 이미지 feature (Feature Enhancer 출력) "내 쿼리가 담당할 bbox 위치가 이미지 어디인가" → 위치 정보 획득
Text Cross-Attention 추가 쿼리 900개 텍스트 feature (BERT 출력) "내 쿼리가 찾는 물체가 텍스트의 어느 단어(cat? dog?)와 대응하는가"

Feature Enhancer와 Decoder의 Cross-Attention — 왜 두 번 하는가

Feature Enhancer에도 Image↔Text Cross-Attention이 있는데, Decoder에서 또 Cross-Attention을 하는 것이 중복처럼 보일 수 있다. 하지만 Q가 누구인지가 완전히 다르다.

Feature Enhancer
Image→Text Cross-Attn		 이미지 feature 전체
(모든 공간 위치)		 이미지 feature 자체를 언어 정보로 강화 강화된 이미지 feature (이미지 feature가 업데이트됨)

Text Cross-Attn		 쿼리 900개

비유로 이해하기
Feature Enhancer = 시험 전 공부 / Decoder = 실제 시험
Feature Enhancer: 이미지 feature와 텍스트 feature가 서로를 참고해 자신을 업데이트한다. "이미지야, 텍스트를 읽고 네 표현을 더 언어 친화적으로 다듬어라." "텍스트야, 이미지를 보고 네 표현을 더 시각 친화적으로 다듬어라." 이 단계가 끝나도 이미지 feature와 텍스트 feature는 여전히 분리된 두 개의 feature다. 서로를 참고해 풍부해졌을 뿐, "이미지의 이 영역 = 텍스트의 이 단어"라는 매핑은 아직 확정되지 않았다.

Cross-Modality Decoder: 쿼리라는 제3의 존재가 등장해 이미지와 텍스트를 동시에 참조하며 "나는 이미지의 이 위치를 봐야 하고, 그것은 텍스트의 'cat'이라는 단어에 해당한다"는 매핑을 레이어마다 반복해서 확정한다. Feature Enhancer 없이 Decoder만 있으면 품질 낮은 재료로 요리하는 것이고, Decoder 없이 Feature Enhancer만 있으면 재료를 잘 다듬었지만 요리를 안 한 것이다.

기법 Sub-Sentence Level 텍스트 표현

여러 카테고리를 한 번에 검출할 때 텍스트 표현 방법이 성능에 큰 영향을 준다. 기존에는 문장 수준(sentence-level)과 단어 수준(word-level) 두 가지 방식이 있었는데, 각각 단점이 있다.

Sub-Sentence Attention Mask
입력 텍스트 "cat . dog . person"에서 "cat" 토큰은 "cat"과 관련된 토큰끼리만 attention이 가능하고, "dog" 토큰은 차단된다. 이 attention mask는 카테고리 구분자(".")를 기준으로 자동 생성된다. 단어 수준의 세밀함을 유지하면서 카테고리 간 노이즈를 제거하는 절충안이다.

실전 활용 라벨 자동 생성기로 사용하기 — 디코더에 대한 오해 정리

Grounding DINO는 파인튜닝 대상 모델이 아니라 라벨 자동 생성기(Auto-Labeler)로 활용되는 경우가 많다. 이 맥락에서 자주 생기는 오해 두 가지를 짚는다.

오해 1 — "디코더는 이미지를 생성할 때 쓰는 것 아닌가?"

혼동의 원인
"인코더-디코더"라는 단어가 생성 모델(VAE, Stable Diffusion)과 검출 모델(DETR, DINO) 양쪽에서 쓰이지만, 역할이 완전히 다르다.
맥락 인코더 역할 디코더 역할 출력
생성 모델
VAE · Diffusion		 이미지 → 잠재 벡터로 압축 잠재 벡터 → 이미지 복원/생성 새로운 이미지
DETR 계열 검출기
DINO · Grounding DINO		 이미지 → feature 추출·강화 쿼리 → bbox 좌표 + 클래스 예측 bbox + 텍스트 구문

Grounding DINO의 디코더는 이미지를 생성하지 않는다. 쿼리 900개를 받아 "이 쿼리가 담당하는 영역의 bbox 좌표와 대응 텍스트 구문"을 출력하는 것이 전부다.

오해 2 — "파인튜닝 안 하면 디코더가 필요 없지 않나?"

혼동의 원인
파인튜닝(Fine-tuning)은 가중치를 업데이트하는 학습 과정이고, 디코더 사용은 추론(inference) 시 모델을 실행하는 것이다. 이 두 개념이 혼동된 경우다.
정확한 이해
파인튜닝을 하지 않는다는 것은 Grounding DINO의 가중치를 업데이트하지 않는다는 뜻이지, 디코더를 사용하지 않는다는 뜻이 아니다. 추론 시에는 인코더와 디코더가 모두 동작해야 bbox가 출력된다. bbox 자체가 디코더의 출력이기 때문이다.
파인튜닝과 추론의 차이
가중치 업데이트 ≠ 모델 실행
파인튜닝: 학습 데이터로 역전파를 실행해 가중치를 수정하는 과정. 추론: 입력을 넣고 출력을 얻는 forward pass만 실행. 라벨 자동 생성은 추론만 하므로 인코더·디코더가 모두 동작한다.
라벨 자동 생성의 장점
수동 라벨링 비용 절감
텍스트 프롬프트("helmet", "safety vest" 등)만 정의하면 수천 장 이미지의 bbox를 자동 생성 가능. 생성된 라벨은 더 가벼운 모델(YOLO 등)의 학습 데이터로 활용한다.
주의사항
자동 생성 라벨의 품질 검수
Grounding DINO가 출력한 bbox가 항상 정확하지는 않다. confidence threshold 조정, 일부 샘플 수동 검수, 데이터 정제 과정이 함께 필요하다.

요약 핵심 요약 — 기여와 한계

기여

Grounding DINO의 세 가지 핵심 기여
① 구조적 원칙 수립: "이미지-언어 feature 융합은 neck · 쿼리 초기화 · head 세 단계 모두에서 수행해야 최고 성능"임을 ablation으로 증명. 단편적 융합에서 전면적 융합으로의 패러다임 전환.

② Transformer의 자연스러운 확장: DINO의 self-attention 구조가 BERT의 그것과 동일함을 활용. 언어 모듈을 구조 변경 없이 cross-attention으로 자연스럽게 통합하는 설계 원칙 제시.

③ REC까지 통합된 단일 프레임워크: Open-set detection과 Reference Expression Comprehension을 동일한 모델로 처리. 텍스트 입력의 형태(카테고리 목록 vs. 속성 설명)에 무관하게 동작.

한계 및 후속 연구

한계
추론 속도
이미지 + 텍스트 두 인코더를 동시에 실행하고 세 단계 fusion을 거치므로, 순수 이미지 기반 검출기보다 추론 속도가 느리다. 실시간 응용에는 최적화 필요.
한계
텍스트 길이 제한
BERT의 최대 토큰 수(512) 제한으로 매우 긴 카테고리 목록이나 복잡한 표현을 한 번에 처리하기 어렵다. 분할 처리 시 성능 저하 가능.
후속 연구
SAM + Grounding DINO
Segment Anything Model(SAM)과 결합하면 텍스트 프롬프트로 임의 물체를 검출하고 픽셀 수준 세그멘테이션까지 수행하는 강력한 파이프라인 구성 가능.

세 단계 Fusion의 연결 고리
Feature Enhancer가 feature를 정렬하고, Language Query가 관련 영역을 선택하며, Cross-Modality Decoder가 매 레이어에서 언어와 이미지를 통합한다
기존 방법이 언어 정보를 neck 한 곳에서만 융합했다면, Grounding DINO는 처음(backbone 출력) 부터 끝(decoder head)까지 일관되게 이미지와 언어를 서로 참조하게 한다. 이 일관된 tight fusion이 zero-shot 성능의 핵심이며, DINO의 Transformer 구조가 이를 자연스럽게 가능하게 한다.
이 논문에서 얻어야 할 한 가지
Grounding DINO의 진짜 기여는 새로운 attention 메커니즘이나 대규모 데이터셋에 있지 않다. "얼마나 자주, 얼마나 많이 이미지와 언어를 서로 참조하게 하는가"가 open-set 검출 성능을 결정한다는 원칙의 실증이다. Transformer 기반 검출기라면 이 원칙을 적용하는 비용이 낮고 효과가 크다는 것도 함께 보여줬다.
