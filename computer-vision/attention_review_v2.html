소개 논문 소개

본 논문이 발표될 때까지만 해도 RNN이나 LSTM과 같은 Recurrent한 구조나 Encoder-Decoder를 포함한 구조를 가진 모델들이 번역 모델에서 큰 성과를 내고 있었습니다. 하지만 이러한 모델들은 큰 Memory와 Computation을 요구하고 있고, 문장의 길이가 길어질수록 한계를 드러내고 있었습니다.

이 문제점을 해결할 방법이 필요했는데, 기존 Encoder-Decoder 구조를 벗어난 Attention Mechanism만으로 구성된 Transformer 모델이 발표되면서 성능과 메모리 두 가지 측면에서 우세한 모델이 등장하게 됩니다. 현재 Transformer는 컴퓨터 비전 분야에도 쓰이고 있으며 많은 변화를 가져온 모델입니다.

기존의 문제
Recurrent 구조의 한계
RNN/LSTM은 이전 hidden state에만 의존하는 순차적 구조로 병렬화가 불가능하며, 시퀀스가 길어질수록 메모리와 성능 한계가 뚜렷합니다.
기존의 문제
Context Vector 병목현상
Seq2Seq의 고정 크기 Context Vector는 긴 문장의 정보를 압축하는 과정에서 병목(Bottleneck)이 발생하여 성능 하락을 유발합니다.
Transformer 해결
Attention Is All You Need
Recurrence와 Convolution을 완전 제거하고 Attention만으로 전체 시퀀스를 병렬 처리해 속도와 성능을 동시에 개선했습니다.

배경 Sequence Modeling — RNN과 Seq2Seq

RNN 모델

기존에 잘 알려진 모델들은 RNN과 LSTM 등이 있습니다. 이 두 모델은 Transformer가 나오기 전까지 실제로 State-Of-The-Art의 성능을 보이고 있었습니다. RNN 모델의 경우 이전 hidden state ht-1과 현재 입력 토큰 xt로 ht를 생성하는 구조입니다.

이러한 모델은 전체 문장의 흐름을 보는 것이 아니라 이전의 hidden state만 보는 순차적인(Locality) 특성으로 문장 길이가 길어지면 batch와 메모리에 한계가 생깁니다. 또한 길이가 다른 시퀀스에서의 예측이 어렵고, 전체 문장 흐름이 아니라 각 단어마다 이전 Hidden State만 보고 계산하는 Locality 문제가 있습니다. 예를 들어 한국어와 영어를 번역하는 RNN 모델이라고 가정하면, 두 언어는 문법적 위치가 다르고 길이도 다르기 때문에 현실적으로 반영하기 어렵습니다.

그림 1. RNN 구조 — 이전 hidden state ht-1과 현재 입력 xt로 ht를 순차 생성

이를 개선하기 위해 제안된 구조가 Encoder-Decoder 구조인 Seq2Seq입니다.

Seq2Seq (Encoder-Decoder)

Encoder에서 입력된 단어의 흐름을 순차적으로 입력받아 이 정보들을 하나의 벡터로 압축하는데 이것을 Context Vector라고 합니다. 이 압축된 Context Vector를 Decoder로 전송하여 번역된 단어를 한 개씩 순차적으로 출력하는 구조입니다.

그림 2. Seq2Seq Encoder-Decoder 구조 — Encoder → Context Vector → Decoder 순차 출력
핵심 한계 — 병목현상
시퀀스 길이마다 입력되어야 할 정보가 다르기 때문에 Context Vector의 크기는 가변적이어야 하는데, Context Vector의 크기는 고정되어 있습니다. 따라서 Encoder에서 시퀀스의 정보를 압축하는 과정에서 병목현상(Bottleneck)이 발생하여 성능 하락의 원인이 됩니다.

이 틀을 깨버린 것이 Transformer입니다. 기존의 Locality 구조가 아니며 병목현상이 발생하지 않는 Attention Mechanism을 사용하여 Global하게 전체 Sequence의 구조를 파악합니다.

그림 3. Transformer 모델 전체 아키텍처 (Vaswani et al., 2017) — Figure 1: The Transformer - model architecture

왼쪽의 회색 큰 박스는 Encoder, 오른쪽의 회색 큰 박스는 Decoder의 구조이고, N개를 쌓아 올려 레이어 층을 깊게 만드는 구조입니다. 각각 Input Embedding과 Positional Encoding을 거쳐 입력되며 Encoder, Decoder에 총 3가지 Attention이 있습니다.

Encoder 구조
Multi-Head Self-Attention + Feed-Forward (N=6)
첫 번째 주황색 Sub-Layer는 Multi-Head Encoder Self-Attention, 파란색 Sub-Layer는 단순한 Feed-Forward입니다. 두 Sub-Layer 각각 Residual Connection을 사용하고 Layer Normalization을 사용합니다.

본 논문에서는 N=6으로 동일한 6개의 층을 쌓았고, 출력 차원은 dmodel = 512로 설정하였습니다. 처음 Encoder에 입력부터 출력까지 총 512차원으로 고정됩니다.
Decoder 구조
Masked MHA + Cross-Attention + Feed-Forward (N=6)
첫 번째 주황색 Sub-Layer는 Masked Multi-Head Decoder Self-Attention, 두 번째 주황색 Sub-Layer는 Encoder 출력을 입력받는 Multi-Head Encoder-Decoder Attention, 파란색 Sub-Layer는 마찬가지로 단순한 Feed-Forward입니다. Residual Connection과 Layer Normalization으로 동일하게 구성됩니다.

핵심 Attention 이란?

Attention이란 같은 문장 내에서 단어들 간의 관계를 나타내며, 이 관계를 Query(Q), Key(K), Value(V) 3가지로 표현하여 하나의 출력 결과를 내는 것입니다.

직관적 이해 — 번역 예시
영어: I love her.  →  한국어: 나는 그녀를 사랑한다.
Q, K, V 역할
"나는"이라는 주체가 Query, 연관성을 찾는 대상인 (I, love, her)이 Key, Q와 K의 유사성을 계산하여 유사한 만큼의 Value값을 가져옵니다. 일반 딕셔너리는 일치하는 K값에 V값을 가져오는 것인데, Attention Mechanism은 Q, K의 유사도 비율만큼 V를 가중합산한다고 보시면 됩니다. Q, K, V는 단순한 스칼라가 아니라 Vector이고, 출력도 마찬가지로 Vector입니다.

아래 그림에서 왼쪽은 한 개의 Attention(Scaled Dot-Product Attention)을 표현한 것이고, 이러한 Attention이 여러 개 모여서 만들어진 것이 오른쪽의 Multi-Head Attention입니다.

그림 5. (좌) Scaled Dot-Product Attention 구조, (우) Multi-Head Attention 구조 — 논문 원본 Figure 2

핵심 Scaled Dot-Product Attention

그림 6. Scaled Dot-Product Attention 연산 흐름도
그림 7. Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V

Q와 K로 Attention Score를 구하고 Q, K, V Dimension(dk)의 루트만큼 나누어 softmax를 취해주고, 이 확률값에 V를 곱하여 구합니다.

예시 설정: Input Embedding 차원 = 5, 각 Head의 차원 = 3 (예시를 위함이며 실제로는 훨씬 많습니다). (I, love, her)과 (나, 사랑, 그녀)로 Tokenize 된다고 가정합니다.

Step 1 — Linear 변환으로 Q, K, V 획득 + MatMul(Q·Kᵀ)

그림 8. Linear 연산으로 Q(3×5), K(5×3) 획득 → 내적으로 Attention Score(3×3) 계산. Score는 두 단어 간 유사도를 의미

Linear 연산을 통해 각각 Q, K, V를 얻게 됩니다. 이후 Q와 K의 내적(MatMul)으로 Attention Score를 얻게 됩니다. Attention Score는 두 단어들 간의 유사도(similarity)를 의미합니다.

Step 2 — Scale(÷√d_k) → Softmax

그림 9. Attention Score를 √d_k로 나눈 뒤 Softmax로 확률값 변환 — softmax(Q·Kᵀ/√d_k)

이후 Attention Score를 dk의 루트로 나누어 Softmax의 확률값을 취해줍니다.

왜 √d_k로 나누는가?
dk 값이 너무 커지게 되면 행렬 연산의 값도 커지면서 softmax 함수가 극도로 작은 기울기를 갖는 포화(saturation) 영역에 빠질 것으로 추측됩니다. √dk로 나누어 정규화하면 기울기 소실 없이 안정적인 학습이 가능합니다.

Step 3 — MatMul with V (최종 Attention 출력)

그림 10. Softmax 확률값에 V를 곱하여 최종 Attention 출력 생성 — Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V

마지막으로 Softmax로 구한 확률값에 V값을 곱하여 Attention 과정을 마무리합니다. 하지만 Transformer는 이러한 Attention 구조가 여러 개인 Multi-Head Attention 구조로 이루어져 있습니다.

핵심 Multi-Head Attention

그림 11. Multi-Head Attention — h개의 Scaled Dot-Product Attention을 병렬 실행 후 Concat → Linear

Input Embedding + Positional Encoding의 차원이 d라고 가정해봅시다. Attention 구조를 한번 실행시켜도 되지만, 굳이 h개로 나누어 d/h의 차원으로 쪼개 병렬연산하여 h개를 학습하고 다시 Concat하고 Linear Layer에 입력합니다.

그림 12. MultiHead(Q,K,V) = Concat(head₁,...,headₕ)W𝑂   where headᵢ = Attention(QWᵢ𝑄, KWᵢ𝐾, VWᵢ𝑉)
핵심 질문
굳이 h개로 나누어야 할 이유가 있는가? 그냥 바로 forwarding하면 안 되는 것인가?
논문의 답변
본 논문에서는 여러 개로 나눈 Multi-Head들이 서로 다른 Representation Subspaces를 학습하여 다양한 유형의 종속성을 가져 다양한 정보를 결합할 수 있다고 제시합니다. 또, 원래 d차원을 h개로 쪼개 병렬연산한 것이기 때문에 총 계산비용은 하나의 head를 사용했을 때와 동일합니다.

구조 Attention 3종류

Transformer Architecture에서 Attention은 Encoder-Decoder Attention, Encoder Self-Attention, Decoder Self-Attention 총 3가지 종류가 있고, 모두 동작 방식이 하나하나 다릅니다.

① Encoder Self-Attention
Q, K, V 모두 Encoder로부터
Encoder의 각 Layer들은 이전 Layer의 출력을 입력으로 받아 이전 Layer에 있는 모든 Position에 대응이 가능합니다.

예시: "I love her"에서 I가 Query일 때 각 I, love, her이 Key로 대응되고, love, her이 Query일 때도 마찬가지로 Attention값을 구합니다.
② Masked Decoder Self-Attention
이후 단어는 참조하지 않음
Auto-Regressive 특성 보존을 위해 이후에 나올 단어들은 참조하지 않습니다. 미래 위치를 -∞로 마스킹합니다.

Q=I → K=[I]
Q=love → K=[I, love]
Q=you → K=[I, love, you]
(미래 단어 참조 = 치팅과 동일)
③ Encoder-Decoder Attention
K, V는 Encoder / Q는 Decoder
Encoder의 마지막 Layer 출력에서 Key, Value를 가져오고, 이전 Decoder에서 Query를 사용합니다.

이렇게 하면 Decoder의 Sequence들이 Encoder의 Sequence들과 어떠한 연관을 가지는지 학습합니다. 기존 Seq2Seq의 병목현상을 해소합니다.

구성 Position-wise Feed-Forward Networks

Encoder와 Decoder 모두 Attention의 sub-layers로 Fully Connected Feed-Forward Network가 포함되어 있습니다 (Transformer Architecture에서 파란색 부분). 이 부분은 2개의 Linear Layer가 이어져 있고, 사이에는 ReLU Activation을 사용하였습니다.

FFN 구조
FFN(x) = max(0, xW₁+b₁)W₂+b₂
모델의 input과 output은 동일하게 dmodel = 512 차원이고, 중간 Inner-Layer의 차원은 dff = 2048로 설정하였습니다. 각 position에 독립적으로 동일한 FFN을 적용합니다.
Add & Norm
LayerNorm(x + Sublayer(x))
각 Sub-Layer마다 Residual Connection을 더하고 Layer Normalization을 적용합니다. 깊은 네트워크에서 기울기 소실을 방지하고 학습을 안정화합니다.

구성 Positional Encoding

Transformer는 LSTM, RNN 기반의 모델처럼 시간의 연속성을 다루지 않습니다. RNN 기반 모델은 구조 자체가 순차적 정보를 자동으로 보장하지만, Transformer는 Attention 함수 기반으로 Q를 기준으로 다른 K들의 연관성만 따지기 때문에 Sequence에 대한 순서 정보가 존재하지 않습니다.

예시: "I love her"을 번역할 때 "사랑해 그녀 나"처럼 순서가 뒤섞여 번역되는 것은 원하는 결과가 아닙니다. 그래서 도입된 기술이 Positional Encoding입니다.

본 논문에서는 sin/cos 곡선의 상대적인 위치 정보를 Embedding과 동일한 차원에 위치시키는 것입니다. Embedding 차원 dmodel과 동일한 차원만큼의 sin/cos 값을 가져와 단순하게 더해줍니다.

그림 14. Positional Encoding 수식 — pos: 시퀀스 내 위치, i: hidden dimension의 위치. 범위 2π ~ 2π×10000

본 논문에서는 Sin/Cos 함수를 사용하였고, 훈련 중에 더 긴 시퀀스 길이를 추정하는 것이 가능하여 선택하였습니다.

그림 15. Positional Encoding 시각화 — 낮은 차원(좌)에서 빠른 진동, 높은 차원(우)에서 완만한 변화. 각 위치가 유일한 인코딩 값을 가짐

결론적으로 Input Embedding, Output Embedding에 위의 -1~1 사이의 Positional Encoding 값을 더해주는 것입니다.

분석 Self-Attention을 사용한 이유

기존에 RNN, LSTM도 있고 Encoder-Decoder 구조인 Seq2Seq도 있는데 왜 Self-Attention을 사용한 Transformer가 더 좋을까요? 이유를 총 3가지로 나누어 설명합니다.

n: Sequence length, d: Representation dimension, k: kernel size of Conv, r: size of neighborhood in restricted self-attention

그림 16. 레이어 타입별 복잡도 비교 — 논문 Table 1 (Complexity per Layer / Sequential Operations / Maximum Path Length)

1. Total Computational Complexity per Layer (레이어당 계산 복잡도)

계산 복잡도
CNN은 RNN보다 무조건 시간복잡도가 높기 때문에 RNN과 비교합니다. n이 d보다 작으면 Self-Attention이, n이 d보다 크면 RNN이 complexity가 낮아집니다. 하지만 영어 token 개수가 8,000개가 넘어갈 때, 이를 잘 표현하려면 d = 256, 512 이상의 높은 차원이 필요합니다. 보통 문장의 길이가 256, 512를 넘어가는 경우가 없기 때문에 Self-Attention이 complexity가 더 작다고 말할 수 있습니다.

2. The Amount of Computation (병렬화 가능량)

순차 연산량
RNN의 경우 레이어마다 모든 토큰들이 input을 순차적으로 총 n개의 Sequence만큼 입력받아 O(n)을 가집니다. 반면 Self-Attention은 모든 position들의 Attention Score 값들을 한 번에 처리할 수 있기 때문에 사실상 O(1)을 가져, 병렬(parallel) 시스템에서 훨씬 유리합니다.

3. Path Length between Long-Range Dependencies (장거리 의존성)

최대 경로 길이
long-range dependencies는 position 간 멀리 떨어져 있는 단어들 간의 의존성으로, 항상 큰 도전 과제였습니다. RNN, LSTM 같은 구조들도 과거의 정보가 마지막까지 전달되지 못하는 현상이 일어났습니다. 이러한 의존성을 학습하는 능력에 영향을 미치는 핵심 요소는 forward / backward signals 간의 path length입니다. 이 path length가 짧을수록 long-range dependencies를 쉽게 학습할 수 있습니다. Input sequence와 Output sequence를 이어주는 징검다리의 개수라고 생각하시면 편합니다.
그림 16. RNN의 long-range dependency — 앞쪽 토큰(진한 색)의 정보가 뒤로 갈수록 희미해지며 O(n)의 Maximum Path Length를 가짐
RNN vs Transformer 비교
이것의 최대 개수를 Maximum Path Length라 하고, RNN은 한 토큰에 하나씩 layer가 붙어 총 O(n)의 길이입니다. 반면 Transformer는 self-attention이 layer를 쌓아 올려 token끼리의 연산을 하는 것이 아니라, 하나의 self-attention의 각 토큰들이 모든 토큰들과 참조하여 연관성을 구하기 때문에 O(1)입니다.

결론
최초로 전체 구조를 Attention으로 바꾼 Transformer
본 논문은 최초로 전체 구조를 Attention으로 바꾼 Transformer 모델을 제시하였습니다. 기존 RNN, Encoder-Decoder 구조를 모두 Multi-Head Self-Attention으로 변경한 것입니다. Translation tasks에서 기존 모델들보다 상당히 빠르고 이전 최고의 모델을 능가하는 state of the art의 성능을 자랑합니다. 이 논문이 나왔을 때 아주 혁신적이었으며, 현재 BERT, GPT, T5, ViT 등 현대 AI의 핵심 아키텍처가 되었습니다.
현대 AI에 미친 영향
Transformer는 현재 컴퓨터 비전 분야에도 사용되고 있으며 많은 변화를 가져온 모델입니다. 이 논문을 시작으로 attention-based models의 연구가 폭발적으로 늘었고, 이미 많은 파생 연구가 진행 중입니다. 2024년 기준 인용수 10만을 훌쩍 넘긴 AI 역사상 가장 영향력 있는 논문 중 하나입니다.
