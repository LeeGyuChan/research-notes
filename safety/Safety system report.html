1 검토 배경 및 목적

본 과제는 물류 및 생산 공정이 혼재된 사업장에서 지게차·작업자 충돌, 작업자 쓰러짐, 위험구역 무단 진입, 공정 이상 등 다양한 위험 상황을 AI 비전 기술 기반으로 실시간 감지·예방하는 시스템 개발을 목적으로 한다.

과제 추진 배경 요약
기존 CCTV 기반 사후 대응 체계는 위험 발생 시 즉각적 대응이 불가능한 구조적 한계를 가진다.
기술연구소는 현장 위험 유형을 재분석하여 구현 가능한 AI 모델 구성과 Edge 기반 운영 아키텍처를 확정한다.
자동 감시 체계 전환을 통해 골든타임 확보 및 Near-miss 데이터 기반 예방 중심 안전관리 체계를 수립한다.

2 현장 문제 재정의

물류·생산 혼재 사업장에서는 작업자, 지게차, 중량물 이동이 동시에 이루어지며 아래 네 가지 위험 상황이 상시 발생 가능한 환경이다. 기존 수동 모니터링 체계로는 발생 즉시 식별·대응이 사실상 불가능하다.

현 상황: 관리자가 CCTV 화면을 직접 모니터링하는 수동 체계에 의존. 다수 카메라 동시 주시 불가 및 인지 지연으로 위험 발생 후 사후 확인에 그침. AI 자동 감지 체계로의 전환이 시급하다.

3 감지 장치 구성 확정

현장 설치 장치는 AI CCTV와 스마트 비전센서 두 종류로 확정한다. AI CCTV는 동선·행동 감지를 담당하고, 비전센서는 공정 판정에 특화하여 역할을 분리한다.

AI CCTV — 현장 전역
동선·행동·구역 감지 전담
작업자·지게차 동선이 교차하는 주요 구역 및 위험구역 경계에 설치.

Edge AI 장치와 직결 — 영상 데이터를 네트워크를 통해 Edge AI 장치로 실시간 전송하여 지연 없는 분석 수행.

ROI는 위험구역 경계선, 지게차-작업자 교차 구간으로 한정하여 처리 부하 최소화.

U-Bolt 벤딩 공정 삽입 지점에 특화 설치.

삽입부 ROI에 집중한 이진(OK/NG) 판정으로 경량 추론 구현.

PLC 인터락 직결 — NG 판정 시 적색 신호등 점등 및 부저 출력으로 작업자의 물리적 작업 진행을 즉시 억제하는 구조적 통제 구현.

AI CCTV 감지 영역과 독립적으로 운영하여 단일 장애가 전체 시스템에 영향을 주지 않도록 설계.

4 모델 아키텍처 설계

위험 유형별로 객체 인식·행동 분석 파이프라인과 공정 판정 파이프라인이 독립적으로 운영되며, 위험 등급 판단 엔진이 두 파이프라인 출력을 종합하여 최종 대응을 결정하는 이중 파이프라인 구조로 설계한다.

객체·행동 파이프라인 — 구성 요소
전처리 ROI 영역 한정 크롭, 프레임 정규화, 카메라별 캘리브레이션
→
객체 인식 (YOLO 계열) Edge 환경 최적화 경량 모델. 작업자·지게차 동시 탐지, 바운딩박스 + 클래스 출력
→
행동 분석 객체 간 거리·이동벡터 계산. Pose Estimation으로 자세 이상 판단
→
위험 이벤트 출력 이벤트 유형·신뢰도 스코어를 판단 엔진으로 전달

경량화 설계 원칙: Edge AI 장치에서 0.5초 이내 추론을 보장하기 위해 YOLOv 계열 경량 모델과 TensorRT 최적화를 적용. 카메라 수 증가에 따른 처리 부하는 ROI 한정 추론으로 완화.

공정 판정 파이프라인 — 구성 요소
전처리 삽입부 ROI 한정 크롭. 조도 변화 대응 정규화. 이진화 전처리.
→
경량 CNN 분류기 OK/NG 이진 분류. 작업 사이클당 1회 추론. 초기 데이터는 OK/NG 이미지 라벨링으로 수집.
→
인터락 출력 NG 판정 즉시 PLC 신호 출력. 적색등·부저 작동. 0.5초 이내 응답 필수.

학습 데이터: 정상(OK) 불량(NG) — 초기 수백 장 수집 후 Data Augmentation 병행. 운영 중 추가 누적.

위험 등급 판단 엔진 — 단계별 운영
이벤트 수신 객체·행동 파이프라인과 공정 판정 파이프라인의 출력을 실시간 수신
→
1단계: Rule 기반 등급 분류 (초기) 이벤트 유형·신뢰도·지속 시간 기반 Rule로 즉시 등급 판정. 구조 단순 — 데이터 부족 초기에도 안정적 운영 가능.
→
2단계: 학습 기반 분류 (데이터 충분 후) 누적 이벤트 데이터로 등급 분류 모델 학습. Rule 기반 폴백은 항상 유지.

5 아키텍처 선택 비교 및 근거

객체 인식 모델은 데이터 효율, 추론 속도, Edge 환경 적합성 4개 기준으로 평가하였다. 공정 판정 모델은 이진 분류 특성상 경량 CNN으로 단독 확정한다.

아키텍처데이터 효율추론 속도Edge 적합성구현 복잡도채택 여부
YOLO 계열 (객체 인식) 높음 매우 빠름 매우 적합 낮음 채택
경량 CNN (공정 판정) 높음 빠름 적합 낮음 채택
Pose Estimation (행동 보조) 중간 중간 적합 중간 병행 채택
Transformer 기반 행동 인식 낮음 중간 검토 필요 높음 3단계 검토
Video Swin Transformer 매우 낮음 느림 부적합 매우 높음 미채택
YOLO 채택 근거 — Edge 실시간 추론 요건
YOLO 계열은 단일 신경망에서 객체 탐지와 클래스 분류를 동시에 처리하는 구조로 추론 속도에서 압도적 우위를 가진다. TensorRT FP16 최적화 적용 시 Edge AI 장치에서 카메라 다수를 동시 처리하면서도 0.5초 이내 응답을 보장할 수 있다. 사전학습 가중치(COCO 등)를 활용한 전이학습으로 초기 데이터 부족 문제도 완화 가능하다.

6 학습 전략 — 단계적 구조 채택

위험 이벤트 데이터의 초기 부족 문제를 해결하기 위해, 사전학습 모델 전이학습으로 시작하여 현장 데이터 누적에 따라 점진적으로 고도화하는 3단계 학습 전략을 채택한다.

1단계 착수~개발 초기
전이학습 기반 빠른 적용
COCO 사전학습 YOLO 가중치를 현장 이미지로 파인튜닝. 라벨 수집 부담을 최소화하면서 작업자·지게차 탐지를 빠르게 구현. 위험구역은 Rule 기반 좌표 판정으로 즉시 적용 가능.
2단계 개발~실증
현장 특화 데이터 누적 및 등급 분류 고도화
위험 이벤트 발생 시마다 라벨 부여하여 현장 특화 학습 데이터 축적. Focal Loss 적용으로 정상(다수)·위험(소수) 클래스 불균형 대응. 위험 등급 3-class 분류 모델로 전환.
3단계 데이터 1만 건 이상
멀티모달 융합 및 MLOps 자동화
복수 카메라 시점 융합 및 Transformer 기반 행동 인식 모델 적용 검토. MLOps 파이프라인으로 이벤트 누적 시 자동 재학습·배포. Near-miss 데이터 기반 예방 인사이트 생성.

7 Edge-서버-관제 아키텍처 설계

추론은 현장 Edge 디바이스에서 전담하여 0.5초 이내 경광등·인터락을 보장하고, 서버는 데이터 적재 및 MLOps 재학습에 집중하며, 관리자는 웹 관제 대시보드로 PC·모바일에서 실시간 모니터링한다.

Edge AI 장치 — Jetson 계열 (Orin Nano 8G 기준)
AI 성능: 20 TOPS · GPU: 512 CUDA core (Ampere) · 메모리: 8GB LPDDR5 통합
인터페이스: USB 3.0 (카메라), RS-485 (비전센서), GigE LAN (PLC·서버), M.2 NVMe SSD
소프트웨어: JetPack 6.x (Ubuntu 22.04) · PyTorch → TensorRT FP16 변환 필수

설계 원칙: 추론은 반드시 Edge에서만 수행. 서버 레이턴시에 의존할 경우 0.5초 인터락 보장 불가. 서버는 데이터 적재와 모델 재학습·배포 역할만 담당.

9 작업자 행동 통제 및 책임 체계

AI 감지 기술에 그치지 않고, 작업자 행동을 구조적으로 제어하는 다층적 운영 방안을 병행 적용하여 현장 실효성을 확보한다.

기술 기반 행동 통제

인터락
시청각 인터락 신호
위험구역 침입 또는 규정 위반 행동 감지 시, 경광등·부저 발령과 동시에 작업자의 동작을 중단시키는 시청각 인터락 신호 출력.
공정 통제
공정 물리적 억제
U-Bolt 삽입 NG 판정 시 적색 신호등 점등 + 부저로 작업자에게 직접 신호하여 물리적 작업 진행을 억제하는 구조적 통제.
팝업 알람
긴급 상황 즉시 대응
긴급 등급 판정 시 관리자 모바일 알림과 동시에 관제 대시보드에 해당 카메라 영상 전면 팝업 즉시 표출.

운영자 및 관리자 대응 책임 체계

역할대응 내용기준 등급
현장 작업자경고 발령 시 즉시 동작 중단 및 안전구역 이동경고 이상
현장 관리자경광등 발령 확인 후 현장 접근, 작업 중지 지시긴급
안전관리 담당자관제 대시보드 모니터링 및 이벤트 기록 확인, 재발 방지 조치전 등급
대표이사·경영진중대사고 임박 알람 수신 시 즉시 보고 체계 가동긴급 반복

현장 운영 루틴: 작업 시작 전 일일 안전 점검(TBM)에서 관제 대시보드의 전일 위험 이벤트 요약을 작업자에게 공유하는 루틴을 수립. 위반 이력은 안전 교육 기초 자료로 활용.

11 리스크 및 대응 방안

리스크대응 방안
위험 이벤트 데이터 부족위험 상황 재현 실험 및 Data Augmentation 병행. 초기에는 COCO 사전학습 전이학습으로 라벨 최소화. 위험구역 침입은 Rule 기반으로 즉시 적용하여 학습 데이터 없이 운영 가능.
객체 인식 오탐·미탐위험구역 Rule 기반 보정 로직 결합. 등급별 신뢰도 임계값을 현장 조건에 맞게 조정하여 오탐률 최소화. 카메라 앵글·조도 최적화로 미탐 방지.
카메라 시야 가림·오염설치 전 사각지대 시뮬레이션 필수. 카메라 정기 점검 루틴 수립. 단일 카메라 장애 시 인접 카메라 영역 확장 폴백 로직 설계.
Edge AI 처리 지연TensorRT FP16 변환 의무화 (메모리 ~50% 절감, 속도 2~3배 향상). ROI 한정 추론으로 피크 부하 분산. 0.5초 이내 응답 지속 모니터링.
현장 작업자 행동 통제 실효성기술 감지와 병행하여 바닥 마킹·안내 표지판·TBM 루틴으로 다층 운영. 위반 이력 관리 기능으로 반복 위반자 집중 교육. 책임 체계 매뉴얼 명문화.
인터락 오작동으로 인한 생산 중단긴급 등급 판정 기준을 보수적으로 설정. 단일 프레임이 아닌 N 프레임 연속 판정으로 순간 노이즈 오탐 방지. 관리자 수동 오버라이드 기능 설계.
